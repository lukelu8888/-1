## 系统概述

### 业务场景
业务员创建采购需求（QR）后，需要等待采购员完成成本核价并反馈。收到采购反馈后，业务员可查看成本信息、采购员的专业建议，并基于此创建客户报价单（QT），完成整个询价-报价闭环。

### 核心价值
✅ **完整闭环** - 客户询价 → QR采购需求 → 采购反馈 → QT客户报价 → SO销售订单
✅ **权限隔离** - 业务员看不到供应商信息，防止跳单
✅ **智能辅助** - 自动填充成本、建议利润率、一键创建报价
✅ **高效协作** - 采购员与业务员无缝对接

### 3️⃣ 创建客户报价（基于采购反馈）

**路径：** `/components/salesperson/CreateQuotationFromFeedback.tsx`

**功能：**
- 自动从采购反馈提取成本信息
- 根据建议利润率自动计算售价
- 业务员可调整售价、利润率
- 自动生成 QT-区域-YYMMDD-编号 格式
- 自动关联 QR、INQ
- 支持批量调整利润率或单独调整单品售价

**智能填充：**
- ✅ 成本单价（从采购反馈）
- ✅ 数量（从采购需求）
- ✅ 利润率（采购员建议值）
- ✅ 销售单价（自动计算 = 成本 × (1 + 利润率)）
- ✅ 付款方式（从采购反馈）
- ✅ 交货条款（从采购反馈）
- ✅ 客户信息（从关联的INQ）

│    ↓                                                             │

│    Admin Portal → 销售订单 → 下推采购订单 → 发给供应商         │

## 使用指南

### 业务员端操作步骤

#### Step 1: 访问"我的采购需求"页面

```typescript
// 在 AdminDashboard.tsx 或业务员 Portal 中添加入口
import { MyPurchaseRequirements } from './components/salesperson/MyPurchaseRequirements';

// 添加导航菜单
<Nav.Link onClick={() => setActiveSection('myPurchaseRequirements')}>
  我的采购需求
</Nav.Link>

// 渲染组件
{activeSection === 'myPurchaseRequirements' && <MyPurchaseRequirements />}
```

#### Step 2: 筛选和搜索

- 点击 **"等待反馈"** 筛选按钮 → 查看还未收到采购反馈的 QR
- 点击 **"已反馈"** 筛选按钮 → 查看已收到采购反馈的 QR
- 在搜索框输入 QR编号、INQ编号或产品名称 → 快速定位

#### Step 3: 查看采购反馈

- 找到状态为 "✅ 已反馈" 的 QR
- 点击 **"查看反馈"** 按钮
- 弹出采购反馈对话框
- 查看：
  - 成本信息汇总（总成本、建议利润率、建议售价、风险评估）
  - 产品成本明细表（成本单价、MOQ、交期）
  - 商务条款（付款方式、交货条款、包装、质保）
  - 采购员专业建议（市场分析、议价空间、风险提示）

#### Step 4: 创建客户报价

- 在采购反馈对话框中，点击 **"基于此成本创建客户报价"** 按钮
- 或者在列表中直接点击 **"创建报价"** 按钮
- 弹出创建报价对话框
- 系统自动填充：
  - 成本单价
  - 建议利润率（默认采购员建议值）
  - 销售单价（自动计算）
  - 客户信息（从关联的 INQ）
  - 商务条款（从采购反馈）

#### Step 5: 调整定价策略

**方式一：批量调整利润率**
- 修改顶部的"利润率"输入框（如从 30% 改为 35%）
- 所有产品的销售单价自动重新计算

**方式二：单独调整单品售价**
- 直接修改表格中某个产品的"销售单价"
- 该产品的利润率自动重新计算
- 总利润和总报价自动更新

#### Step 6: 确认并提交

- 检查客户信息是否正确
- 检查商务条款是否合理
- 添加备注（可选，给客户的特殊说明）
- 点击 **"创建客户报价单"** 按钮
- 系统生成 QT 编号（如 `QT-NA-251221-8888`）
- 报价单保存到数据库
- 可在"我的报价"模块查看和管理
