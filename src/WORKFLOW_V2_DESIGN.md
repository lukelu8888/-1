# 📊 订单流程文档管理 V2 - 极致紧凑版设计说明

## 🎯 V2版本核心改进

### **问题1：订单选择区域单调、无层次感**
❌ **旧版问题：**
- 下拉选择器，只能看到当前选中的订单
- 无法快速对比多个订单的状态
- 缺乏视觉冲击力和专业感

✅ **V2解决方案：横向滚动订单卡片**
```
┌─────┬─────┬─────┬─────┬─────┐
│订单1 │订单2 │订单3 │订单4 │订单5 │ ← 横向滚动
│ ● PO│ ● PO│ ● PO│ ● PO│ ● PO│
│$125K│$89K │$156K│$78K │$92K │
│ 70% │ 85% │ 45% │100% │ 60% │
│剩5天│剩2天│剩10天│已完成│剩7天│
└─────┴─────┴─────┴─────┴─────┘
```

### **问题2：阶段区域padding过大、字体偏大**
❌ **旧版问题：**
- py-2（8px padding）太松散
- text-xs（12px）字体偏大
- 浪费垂直空间

✅ **V2解决方案：超紧凑设计**
- py-1（4px padding）→ 节省50%空间
- text-[10px]（10px）→ 更高信息密度
- 表格行高 py-1.5（6px）→ 更紧凑

## 🎨 顶部订单卡片设计（V2核心亮点）

### **1. 横向滚动卡片列表**
```tsx
<div className="flex gap-2 overflow-x-auto">
  {orders.map(order => (
    <div className="flex-shrink-0 w-52 bg-white border-2 rounded-lg">
      {/* 订单卡片内容 */}
    </div>
  ))}
</div>
```

### **2. 单个订单卡片结构**
```
┌─────────────────────────────────┐
│ ● PO-2024-155     [含税13%]     │ ← 状态灯+订单号+标签
├─────────────────────────────────┤
│ 订单金额 Amount                  │
│ $125,000                        │ ← 大号金额（重点）
├─────────────────────────────────┤
│ 交期: 2024-12-20    剩余: 5天   │ ← 交期+倒计时
├─────────────────────────────────┤
│ 进度: [████████░] 70%          │ ← 进度条+百分比
├─────────────────────────────────┤
│ [进行中]                        │ ← 状态标签
└─────────────────────────────────┘

尺寸：w-52 (208px宽) × auto高
```

### **3. 状态灯颜色系统**
| 状态 | 颜色 | 动画 | 说明 |
|------|------|------|------|
| **已完成** | 🟢 绿色 | 无 | 订单100%完成 |
| **进行中** | 🔵 蓝色 | animate-pulse | 正常进行中 |
| **紧急** | 🟠 橙色 | animate-pulse | 交期临近 |
| **延期** | 🔴 红色 | animate-pulse | 已经延期 |

### **4. 选中状态高亮**
```tsx
// 选中订单：蓝色2px边框 + 阴影
className={`border-2 ${
  isSelected ? 'border-blue-500 shadow-md' : 'border-gray-300'
}`}
```

### **5. 交期倒计时颜色**
```tsx
{
  daysRemaining <= 3 ? 'text-red-600' :    // ≤3天：红色（紧急）
  daysRemaining <= 7 ? 'text-orange-600' : // ≤7天：橙色（注意）
  'text-green-600'                         // >7天：绿色（正常）
}
```

### **6. 进度条颜色**
```tsx
{
  progress === 100 ? 'bg-green-600' :   // 100%：绿色
  progress >= 70 ? 'bg-blue-600' :      // ≥70%：蓝色
  progress >= 40 ? 'bg-orange-500' :    // ≥40%：橙色
  'bg-red-500'                          // <40%：红色
}
```

## 📊 订单卡片信息层级

### **优先级1：订单号 + 状态灯**
```tsx
<div className="flex items-center gap-1.5">
  <div className="w-2 h-2 rounded-full bg-blue-500 animate-pulse" />
  <span className="font-bold text-xs">PO-2024-155</span>
</div>
```
- **最重要** - 订单标识
- 状态灯动画吸引注意

### **优先级2：订单金额**
```tsx
<p className="font-bold text-blue-600 text-sm">$125,000</p>
```
- **次重要** - 订单价值
- 蓝色突出，大号字体

### **优先级3：交期倒计时**
```tsx
<p className="text-xs font-bold text-red-600">5天</p>
```
- **紧急程度** - 时间压力
- 颜色编码（红/橙/绿）

### **优先级4：进度条**
```tsx
<div className="w-full bg-gray-200 rounded-full h-1.5">
  <div className="h-1.5 rounded-full bg-blue-600" style={{ width: '70%' }} />
</div>
```
- **完成度** - 视觉化进度
- 颜色随进度变化

### **优先级5：状态标签**
```tsx
<Badge className="bg-blue-50 text-blue-700">进行中</Badge>
```
- **当前状态** - 文字说明
- 底部总结
