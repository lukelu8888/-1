# 📄 订单流程化文档中心 - 使用说明

## 🎯 功能概述

**订单流程化文档中心**是供应商Portal的核心文档管理系统，以订单为主线，按照B2B外贸业务流程的16个关键节点归类展示和管理文档。

---

## 🔄 16个业务流程节点

### **阶段一：订单前期阶段**

#### ① 询价阶段
- **关键文档**：客户询价单、初步报价单、技术规格书
- **责任方**：客户 + 业务员
- **Admin互动**：无

#### ② 订单确认
- **关键文档**：采购订单PO（核心文档）、订单确认函、合同
- **责任方**：客户 + 业务员
- **Admin互动**：无

#### ③ 定金确认 🔗
- **关键文档**：定金发票、收款水单
- **责任方**：供应商财务
- **Admin互动**：✅ 财务确认收款（30%定金）
- **互动类型**：financial_confirm

---

### **阶段二：样品确认阶段**

#### ④ 产前样确认 🔗
- **关键文档**：产前样照片、产前样报告、快递单号、COSUN确认函
- **责任方**：供应商 + COSUN业务员
- **Admin互动**：✅ 业务员批准生产
- **互动类型**：business_approve

#### ⑤ 生产准备
- **关键文档**：生产通知单、物料清单BOM、技术图纸、供应商询价单
- **责任方**：供应商
- **Admin互动**：无

#### ⑥ 大货样确认 🔗
- **关键文档**：大货样照片、大货样检验报告、COSUN确认函
- **责任方**：供应商 + COSUN业务员
- **Admin互动**：✅ 业务员批准量产
- **互动类型**：business_approve

---

### **阶段三：生产执行阶段**

#### ⑦ 生产过程
- **关键文档**：生产照片（开工/组装/半成品）、生产进度周报
- **责任方**：供应商
- **Admin互动**：无

#### ⑧ 生产质检 🔗
- **关键文档**：内部质检报告（原料/过程/成品）
- **责任方**：供应商 + COSUN QC
- **Admin互动**：✅ QC部门审核质检报告
- **互动类型**：qc_review

---

### **阶段四：验货与包装阶段**

#### ⑨ 验货准备 🔗
- **关键文档**：验货通知函、验货排期表
- **责任方**：供应商 + COSUN单证员
- **Admin互动**：✅ 单证员安排验货时间
- **互动类型**：arrange_inspection

#### ⑩ 验货实施 🔗
- **关键文档**：验货报告、验货照片、验货合格证明
- **责任方**：COSUN验货员或第三方机构
- **Admin互动**：✅ 验货员执行验货
- **互动类型**：inspection_execute

#### ⑪ 包装完成
- **关键文档**：包装照片、装箱清单、唛头照片、装柜照片
- **责任方**：供应商
- **Admin互动**：无

---

### **阶段五：发货与收款阶段**

#### ⑫ 发货准备（工厂→港口/仓库）
- **关键文档**：送货单、物流单号、发货照片、仓库入库单
- **责任方**：供应商
- **Admin互动**：无
- **特别说明**：⚠️ 商业发票、提单由COSUN单证员制作

#### ⑬ 余款确认 🔗
- **关键文档**：余款发票、收款水单
- **责任方**：供应商财务 + COSUN财务
- **Admin互动**：✅ 财务确认付款（70%余款）
- **互动类型**：financial_confirm

#### ⑭ 出库发运 🔗
- **关键文档**：出库通知、货运订舱单、报关单据
- **责任方**：供应商 + COSUN单证员
- **Admin互动**：✅ 单证员制作商业发票、提单、原产地证明
- **互动类型**：documentation
- **特别说明**：COSUN单证员负责制作：
  - Commercial Invoice（商业发票）
  - Bill of Lading（提单）
  - Certificate of Origin（原产地证明）

#### ⑮ 增值税发票 🔗 **（仅含税订单）**
- **关键文档**：增值税专用发票、报关单（出口）、退税联
- **责任方**：供应商财务 + COSUN财务
- **Admin互动**：✅ 财务用于出口退税申报
- **互动类型**：financial_confirm
- **特别说明**：
  - ✅ **含税订单**：必须上传增值税发票
  - ⭕ **不含税订单**：自动跳过此节点
  - ⚠️ 报关后30天内必须开具
  - 💰 用于COSUN申请出口退税

#### ⑯ 交付完成
- **关键文档**：客户签收单POD、售后服务卡、订单结案报告
- **责任方**：客户 + 业务员
- **Admin互动**：无

## 💰 含税 vs 不含税订单

### **含税订单（13% VAT）**
```
订单金额（含税）：$125,000
不含税金额：$110,619
增值税额：$14,381 (13%)

✅ 需要完成"增值税发票"节点
✅ 供应商需开具增值税专用发票给COSUN
✅ COSUN使用发票申请出口退税
```

### **不含税订单**
```
订单金额：$89,000
税额：$0

⭕ 自动跳过"增值税发票"节点
⭕ 无需开具增值税发票
⭕ 流程节点从16个减少到15个
```

## 📊 使用流程

### **1. 选择订单**
```
文档中心顶部 → 选择订单下拉框 → 选择具体订单
例如：PO-2024-155 - COSUN (ABC Corp) - $125,000
```

### **2. 查看订单信息**
```
✅ 订单类型：含税订单 (13% VAT) / 不含税订单
📅 下单日期：2024-11-10
📅 交期日期：2024-12-20
📊 订单进度：[███████░] 70% (11/16 节点完成)
```

### **3. 浏览流程节点**
```
垂直时间线展示16个节点
每个节点显示：
  - 节点序号 + 名称
  - 节点状态（已完成/进行中/待处理）
  - 文档数量
  - Admin互动状态（如有）
```

### **4. 展开节点查看文档**
```
点击节点 → 展开文档列表
显示：
  - 文档名称
  - 上传者
  - 上传日期
  - 文件大小
  - 操作按钮（预览/下载）
```

### **5. 上传文档到节点**
```
展开节点 → 点击"上传文档到此节点"
选择文件 → 填写备注 → 确认上传
✅ 文档立即添加到该节点
✅ 节点状态自动更新
```

### **6. 上传增值税发票（含税订单）**
```
找到"⑮ 增值税发票"节点 → 点击"上传文档到此节点"
自动打开增值税发票专用上传对话框
填写发票信息：
  - 发票代码（12位）
  - 发票号码（8位）
  - 开票日期
  - 税率（13%）
上传扫描件：
  - 增值税专用发票.pdf
  - 报关单（出口）.pdf
  - 退税联.pdf
提交 → 发票信息发送给COSUN财务
```

### **7. 查看Admin端状态**
```
节点展开后，底部显示：
🔗 Admin端状态：
  ✅ 财务（王芳）- 已完成于 2024-11-12
  ⏱ 业务员（李业务员）- 处理中...
  ⚠️ 单证员（张三）- 等待处理
```

## 🔧 技术实现

### **数据结构**
```typescript
interface WorkflowStage {
  id: number;                    // 节点ID (1-16)
  name: string;                  // 节点名称
  key: string;                   // 节点标识
  phase: string;                 // 所属阶段
  status: 'completed' | 'in_progress' | 'pending' | 'blocked';
  requiresAdminInteraction: boolean;  // 是否需要Admin互动
  adminInteractionType?: string;      // 互动类型
  adminStatus?: {                     // Admin端状态
    status: string;
    responsibleRole: string;
    responsiblePerson: string;
    actionDate?: string;
  };
  documents: OrderDocument[];         // 文档列表
  isTaxableOnly?: boolean;            // 仅含税订单需要
}
```

### **核心功能**
- `toggleStage()` - 展开/折叠节点
- `toggleAllStages()` - 全部展开/折叠
- `handleUploadToStage()` - 上传文档到指定节点
- `handleUploadVATInvoice()` - 上传增值税发票
- `calculateProgress()` - 计算订单整体进度
