# 业务员待办中心 V1 字段清单 / 数据来源 / 动作回写表

## 一、目的

这份文档用于把业务员待办中心 V1 的三件事一次性钉死：

1. 页面到底看什么字段
2. 每个字段从哪张真实业务表来
3. 每个动作点下去以后写回哪里

本文档只服务于：

- 业务员待办中心

不适用于：

- 业务员工作台
- 财务待办
- 单证待办
- 采购待办

---

## 二、页面结构范围

V1 仅覆盖以下页面结构：

1. 顶部动作条
2. 主执行表
3. 右侧辅助栏
4. 详情抽屉

## 五、动作回写表

## 5.1 动作总原则

所有动作都必须至少写两层：

1. 写入跟进日志
2. 如命中真实业务节点，则同步回写业务表

不得只做前端状态切换。

---

## 六、V1 最小落地范围

V1 只强制落以下链路：

1. `ING`
2. `QT`
3. `SC`
4. 客户付款跟进
5. 到港确认
6. 收货确认
7. 客户反馈

其他如：

- 第三方验货
- 第三方监装
- 清关资料协同
- 客诉协调

可作为 V1.1 / V1.2 增补。

---

## 七、最终交付判断

只有当业务员待办中心同时做到以下 4 点，才算真正成立：

1. 主表按客户 + 当前业务单组织
2. 节点来自真实业务链，不是抽象手工任务
3. 行内动作能写跟进日志
4. 关键动作能回写真实业务状态

只做到展示，不算完成。
