## 🔗 **五、业务流程集成**

### **事件驱动架构**

#### **已监听的8种业务事件**
1. `CONTRACT_SIGNED` - 合同签订
2. `PROCUREMENT_PAID` - 采购付款
3. `SHIPPING_NOTICE_ISSUED` - 出货通知
4. `CUSTOMS_CLEARED` - 报关完成
5. `VESSEL_DEPARTED` - 开船
6. `PAYMENT_RECEIVED` - 收汇确认
7. `FX_SETTLED` - 结汇完成
8. `DOCS_READY_FOR_TAX_REFUND` - 单证齐全可退税

#### **事件响应示例**
```typescript
// 合同签订 → 创建13项单证任务
useBusinessEvent('CONTRACT_SIGNED', (payload) => {
  console.log('📋 [单证系统UI] 检测到合同签订，刷新订单列表...', payload);
  toast.info(`订单 ${payload.orderId} 的单证任务已创建`);
});

// 单证齐全 → 通知可退税
useBusinessEvent('DOCS_READY_FOR_TAX_REFUND', (payload) => {
  console.log('📋 [单证系统UI] 检测到单证齐全，可提交退税...', payload);
  toast.success(`🎉 订单 ${payload.orderId} 的13项单证已齐全，可提交退税申报！`);
});
```

## 📊 **六、数据结构**

### **ShipmentOrder 接口**
```typescript
interface ShipmentOrder {
  orderId: string;                    // 订单号
  contractNo: string;                 // 合同号
  customerName: string;               // 客户名称
  salesRep: string;                   // 业务员
  businessStage: string;              // 业务阶段
  status: OrderStatus;                // 订单状态
  priority: Priority;                 // 优先级
  completionRate: number;             // 完成率
  requiredDocs: number;               // 必需单证数
  completedDocs: number;              // 已完成数
  missingDocs: number;                // 缺失数
  overdueItems: number;               // 超期数
  
  // 业务日期
  contractDate: string;
  shipmentDate: string;
  customsDate?: string;
  collectionDate?: string;
  settlementDate?: string;
  
  // 贸易信息
  incoterm: 'EXW' | 'FCA' | 'FOB' | 'CFR' | 'CIF';
  paymentTerm: string;
  destPort: string;
  container: string;
  totalValue: number;
  currency: string;
  
  // 单证实例
  documents: { [key: string]: DocumentInstance };
  
  // 预警
  alerts: Alert[];
}
```

### **DocumentInstance 接口**
```typescript
interface DocumentInstance {
  docId: string;
  status: DocStatus;
  uploadDate?: string;
  uploadBy?: string;
  reviewer?: string;
  version: number;
  files: FileInfo[];
  urgency?: 'critical' | 'high' | 'medium' | 'low';
  remarks?: string;
  history: HistoryEntry[];
  deadline?: string;
  daysRemaining?: number;
  blockedBy?: string[];
  qualityScore?: number;
}
```

## 🚀 **七、使用指南**

### **业务员日常操作流程**

#### **1. 查看待办任务**
```
登录 → Admin Portal → 单证管理系统 Ultimate 终极版 → 工作台
→ 查看"当前任务"卡片和"严重预警"面板
```

#### **2. 处理超期单证**
```
工作台 → 找到超期订单（红色背景）→ 点击展开
→ 查看13项单证网格 → 点击超期单证的"上传"按钮
→ 上传文件 → 提交审核
```

#### **3. 查看单个订单详情**
```
工作台 → 点击订单行 → 点击"进入详情"按钮
→ 查看订单概览 → 业务时间线 → 单证网格视图
→ 切换到"甘特图时间线"查看进度
→ 切换到"依赖关系图"查看阻塞节点
→ 在右侧协作面板添加评论和待办任务
```

#### **4. 搜索历史单证**
```
工作台 → 点击"文档库"Tab
→ 使用搜索框输入关键词（订单号/客户名）
→ 或使用高级筛选（时间/区域/质量）
→ 切换到"按单证类型"查看所有D01退税申报表
→ 点击"打包下载"批量导出
```

#### **5. 查看业务流程集成**
```
工作台 → 点击"查看集成架构"按钮
→ 查看5个模块和8种业务事件
→ 了解单证系统如何与其他模块对接
→ 点击"集成指南"查看详细文档
```
