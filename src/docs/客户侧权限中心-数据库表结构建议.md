# 客户侧权限中心数据库表结构建议

## 一、目标

本文用于为 **客户侧 Permission Center** 提供可实施的数据库落库建议。

目标：

- 与我方侧权限中心保持同一套分层思路
- 支持客户侧独立角色模板、模块树、动作权限、数据范围、例外授权与变更日志
- 保证租户隔离
- 为后续多语言展示提供基础字段

---

## 二、设计原则

### 1. 与我方侧共享 RBAC 分层，不共享业务基线

客户侧与我方侧都采用：

- 角色
- 模块
- 菜单可见性
- 动作权限
- 数据范围
- 用户例外授权
- 变更日志

但以下内容必须独立：

- 角色模板
- 模块树
- 数据范围语义
- 默认矩阵

### 2. 所有客户侧权限都必须带租户边界

客户侧权限数据必须与 `tenant_id` 或等价客户组织标识绑定。

### 3. locale 不建议存成权限主数据的一部分

权限本身以稳定 code 存储，展示语言由前端 locale + 翻译资源控制。  
数据库只存必要的默认展示名或 fallback 文案。

### 4. `Purchaser` + `My Products` 需要写入初始化基线

这不是 UI 层临时逻辑，而应在初始化数据或系统基线中写死。

---

## 三、建议表结构总览

客户侧权限中心建议新增或派生以下表：

1. `customer_permission_roles`
2. `customer_permission_modules`
3. `customer_role_module_access`
4. `customer_role_module_actions`
5. `customer_role_module_scopes`
6. `customer_user_permission_exceptions`
7. `customer_permission_change_logs`

如后续希望与我方侧统一表结构，也可采用共享大表 + `portal_type='customer'` 的方式，但第一版更建议独立命名，降低混淆。

---

## 四、各表建议

## 4.1 `customer_permission_roles`

### 作用

客户侧角色模板字典。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `role_code` TEXT NOT NULL
- `role_name_en` TEXT NOT NULL
- `role_name_es` TEXT NULL
- `role_name_pt` TEXT NULL
- `role_name_ar` TEXT NULL
- `role_category` TEXT NOT NULL
- `special_control_code` TEXT NOT NULL
- `responsibility_en` TEXT NOT NULL
- `responsibility_es` TEXT NULL
- `responsibility_pt` TEXT NULL
- `responsibility_ar` TEXT NULL
- `is_system_template` BOOLEAN NOT NULL DEFAULT true
- `is_editable` BOOLEAN NOT NULL DEFAULT true
- `status` TEXT NOT NULL DEFAULT `active`
- `sort_order` INTEGER NOT NULL DEFAULT 0
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- UNIQUE: `(tenant_id, role_code)`
- CHECK `role_category` IN
  - `management`
  - `administration`
  - `business`
  - `support`
  - `operations`
  - `read_only`
- CHECK `special_control_code` IN
  - `all_company_data`
  - `department_scope`
  - `assigned_or_self`
  - `financial_access`
  - `assigned_data`
  - `read_only_scope`
- CHECK `status` IN
  - `active`
  - `inactive`

### 初始化建议

默认角色写入：

- `Company_Owner`
- `Company_Admin`
- `Purchase_Manager`
- `Purchaser`
- `Finance`
- `Warehouse_Receiver`
- `Viewer`

---

## 4.2 `customer_permission_modules`

### 作用

客户侧权限中心识别的模块字典。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `module_code` TEXT NOT NULL
- `module_name_en` TEXT NOT NULL
- `module_name_es` TEXT NULL
- `module_name_pt` TEXT NULL
- `module_name_ar` TEXT NULL
- `module_group_code` TEXT NOT NULL
- `parent_module_code` TEXT NULL
- `module_level` INTEGER NOT NULL DEFAULT 1
- `route_key` TEXT NULL
- `status` TEXT NOT NULL DEFAULT `active`
- `sort_order` INTEGER NOT NULL DEFAULT 0
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- UNIQUE: `(tenant_id, module_code)`
- CHECK `module_group_code` IN
  - `core`
  - `business`
  - `administration`
- CHECK `status` IN
  - `active`
  - `inactive`

### 初始化建议

至少写入：

#### core
- `dashboard`
- `message-center`
- `document-center`
- `company-profile`
- `member-directory`

#### business
- `my-products`
- `inquiry-center`
- `order-center`
- `contract-center`
- `invoice-statement-center`
- `shipment-receiving`
- `after-sales-service`

#### administration
- `permission-center`
- `approval-center`
- `operation-logs`

---

## 4.3 `customer_role_module_access`

### 作用

记录客户侧角色对模块入口的菜单可见性。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `role_id` UUID NOT NULL
- `module_id` UUID NOT NULL
- `is_visible` BOOLEAN NOT NULL DEFAULT false
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- UNIQUE: `(tenant_id, role_id, module_id)`
- FK `role_id` -> `customer_permission_roles.id`
- FK `module_id` -> `customer_permission_modules.id`

### 初始化要求

必须保证：

- `Purchase_Manager` + `my-products` = `true`
- `Purchaser` + `my-products` = `true`

---

## 4.4 `customer_role_module_actions`

### 作用

记录角色在某模块内的动作权限。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `role_id` UUID NOT NULL
- `module_id` UUID NOT NULL
- `action_code` TEXT NOT NULL
- `is_allowed` BOOLEAN NOT NULL DEFAULT false
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- UNIQUE: `(tenant_id, role_id, module_id, action_code)`
- CHECK `action_code` IN
  - `view`
  - `create`
  - `edit`
  - `delete`
  - `approve`

### 初始化要求

对于 `my-products`：

- `Purchase_Manager`:
  - `view = true`
  - `create = true`
  - `edit = true`
  - `approve = true`
- `Purchaser`:
  - `view = true`
  - `create = true`
  - `edit = true`
  - `delete = false`
  - `approve = false`

---

## 4.5 `customer_role_module_scopes`

### 作用

记录角色在某模块内的数据范围。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `role_id` UUID NOT NULL
- `module_id` UUID NOT NULL
- `scope_code` TEXT NOT NULL
- `scope_rules` JSONB NOT NULL DEFAULT `'{}'::jsonb`
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- UNIQUE: `(tenant_id, role_id, module_id)`
- CHECK `scope_code` IN
  - `all_company`
  - `department`
  - `department_and_sub`
  - `self`
  - `assigned`

### `scope_rules` 用途建议

只用于扩展，不要承载主结构。  
典型内容：

- 指定部门 id 列表
- 指定项目 id 列表
- 排除规则

### 初始化建议

- `Company_Owner` -> `all_company`
- `Company_Admin` -> `all_company`
- `Purchase_Manager` -> `department_and_sub`
- `Purchaser` -> `self`
- `Warehouse_Receiver` -> `assigned`

---

## 4.6 `customer_user_permission_exceptions`

### 作用

记录用户级例外授权。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `target_user_id` UUID NOT NULL
- `base_role_id` UUID NOT NULL
- `target_module_id` UUID NOT NULL
- `exception_type_code` TEXT NOT NULL
- `action_overrides` JSONB NOT NULL DEFAULT `'{}'::jsonb`
- `scope_override_code` TEXT NULL
- `scope_override_rules` JSONB NOT NULL DEFAULT `'{}'::jsonb`
- `effective_from` TIMESTAMPTZ NULL
- `effective_to` TIMESTAMPTZ NULL
- `reason` TEXT NULL
- `status` TEXT NOT NULL DEFAULT `active`
- `created_by` UUID NULL
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- CHECK `exception_type_code` IN
  - `temporary_module_access`
  - `temporary_approval_permission`
  - `temporary_data_scope_expansion`
  - `acting_on_behalf`
  - `emergency_business_access`
- CHECK `scope_override_code` IN
  - `all_company`
  - `department`
  - `department_and_sub`
  - `self`
  - `assigned`
- CHECK `status` IN
  - `active`
  - `revoked`
  - `expired`

### 设计建议

- `action_overrides` 可直接存 `{ "view": true, "edit": true }`
- 例外授权只适合临时能力，不应替代主角色矩阵

---

## 4.7 `customer_permission_change_logs`

### 作用

记录客户侧权限中心的配置变更和发布动作。

### 建议字段

- `id` UUID PK
- `tenant_id` UUID NOT NULL
- `operator_user_id` UUID NULL
- `change_type_code` TEXT NOT NULL
- `target_type` TEXT NOT NULL
- `target_code` TEXT NULL
- `before_snapshot` JSONB NOT NULL DEFAULT `'{}'::jsonb`
- `after_snapshot` JSONB NOT NULL DEFAULT `'{}'::jsonb`
- `change_summary` TEXT NULL
- `published_batch_id` UUID NULL
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()

### 约束建议

- CHECK `change_type_code` IN
  - `role_created`
  - `role_updated`
  - `role_deleted`
  - `menu_permission_changed`
  - `action_permission_changed`
  - `data_scope_changed`
  - `exception_authorization_added`
  - `exception_authorization_updated`
  - `exception_authorization_revoked`
  - `changes_published`
- CHECK `target_type` IN
  - `role`
  - `module`
  - `menu_matrix`
  - `action_matrix`
  - `scope_matrix`
  - `exception`
  - `publish_batch`

## 五、推荐索引

建议至少创建以下索引：

- `idx_customer_permission_roles_tenant_status`
- `idx_customer_permission_modules_tenant_group`
- `idx_customer_role_module_access_tenant_role`
- `idx_customer_role_module_actions_tenant_role_module`
- `idx_customer_role_module_scopes_tenant_role_module`
- `idx_customer_user_permission_exceptions_tenant_user_status`
- `idx_customer_permission_change_logs_tenant_created_at`
- `idx_customer_permission_change_logs_tenant_type`

---

## 六、初始化数据建议

初始化时应一并写入：

1. 默认角色模板
2. 默认模块树
3. 默认菜单矩阵
4. 默认动作矩阵
5. 默认数据范围矩阵

### 特别要求

初始化数据中必须显式写入以下事实：

- `Purchaser` 可见 `my-products`
- `Purchaser` 对 `my-products` 拥有 `view/create/edit`
- `Purchase_Manager` 对 `my-products` 拥有 `approve`

---

## 七、共享表 vs 独立表建议

### 第一阶段建议

优先独立表：

- 语义更清楚
- 避免和我方侧角色混表
- 降低错配与误授权风险

### 后续如要统一

可以收敛为共享大表，并补以下字段：

- `portal_type` = `internal` / `customer`
- `tenant_id`

但这一步不建议在第一版同时做。

---

## 八、最终建议

客户侧权限中心数据库层最重要的不是“能不能做出矩阵”，而是：

1. 是否有租户隔离
2. 是否把客户侧角色模板和我方侧模板隔离
3. 是否把 `Purchaser` + `My Products` 基线固化
4. 是否能审计所有发布动作
