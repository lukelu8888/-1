# 客户侧权限中心前端组件复用清单

## 一、目标

本文用于明确客户侧 Permission Center 在前端实现时，哪些组件应直接复用我方侧现有实现，哪些部分只替换数据与文案，哪些地方需要增加客户侧适配层。

目标：

- 保持与我方侧权限中心 **同 UI 布局**
- 最大化复用现有前端结构
- 将客户侧差异收敛到数据、文案、locale 和权限基线上

---

## 二、复用总原则

1. 优先复用现有权限中心页面骨架
2. 优先复用现有 tab 级组件
3. 不复制一套新的视觉结构
4. 客户侧差异尽量通过 props、adapter、copy map 和 service 注入完成
5. Arabic RTL 适配优先通过页面级 direction 和 class 控制，不重写结构

## 四、建议复用结构

## 4.1 页面级容器

### 建议复用

- `PermissionCenterV1.tsx` 的整体页面骨架

### 复用内容

- 顶部标题区
- 顶部按钮区
- 页签导航区
- 主内容区域的版式
- 发布、重置这类全局操作流转

### 建议新增

- `CustomerPermissionCenter.tsx`

### 作用

作为客户侧容器页，负责：

- 接收客户侧 locale
- 读取客户侧 copy
- 读取客户侧角色与模块基线
- 注入客户侧 service
- 复用现有 tab 组件或其抽象版本

---

## 4.2 Tab 级组件

### 可直接复用的结构

| 现有组件 | 客户侧建议 |
|---|---|
| `RoleOverviewTab.tsx` | 复用布局，替换角色字典、列文案、筛选文案 |
| `MenuPermissionMatrixTab.tsx` | 复用布局，替换模块树、列角色、矩阵数据 |
| `ActionsDataScopeTab.tsx` | 复用三栏布局，替换模块树、动作矩阵数据、右栏范围文案 |
| `UserExceptionGrantsTab.tsx` | 复用布局，替换 notice、表头、例外类型 |
| `ChangeLogTab.tsx` | 复用布局，替换筛选文案、日志类型文案 |

### 建议方式

不要复制成一套 `CustomerRoleOverviewTab.tsx`、`CustomerMenuPermissionMatrixTab.tsx` 的完全分叉版本，优先做成：

- 同一个 tab 组件
- 通过 `mode="internal" | "customer"` 分支
- 或通过 adapter / config 注入数据

## 五、建议新增的适配层

## 5.1 Copy Adapter

建议新增：

- `src/components/dashboard/customer-permission-center/customerPermissionCenterCopy.ts`

### 作用

集中管理客户侧 Permission Center 的文案映射。

### 建议输出

- `getCustomerPermissionCenterCopy(locale)`

### 覆盖内容

- page title
- subtitle
- tab labels
- table headers
- buttons
- filters
- empty states
- notice 文案
- scope labels

---

## 5.2 Config Adapter

建议新增：

- `src/components/dashboard/customer-permission-center/customerPermissionCenterConfig.ts`

### 作用

集中管理客户侧角色模板、模块树、默认权限矩阵、数据范围基线。

### 建议输出

- `CUSTOMER_PERMISSION_ROLES`
- `CUSTOMER_PERMISSION_MODULES`
- `CUSTOMER_MENU_MATRIX_BASELINE`
- `CUSTOMER_ACTION_MATRIX_BASELINE`
- `CUSTOMER_SCOPE_MATRIX_BASELINE`

---

## 5.3 Service Adapter

建议新增：

- `src/lib/services/customerPermissionCenterService.ts`

### 作用

将客户侧权限中心从我方侧 `permissionCenterService.ts` 中隔离出来。

### 原则

- 不直接复用我方侧正式角色清单
- 不直接复用我方侧模块字典
- 可以复用相同的数据结构与方法签名

---

## 六、页面组件拆分建议

## 6.1 顶部 Header

### 建议复用

- 现有 `PermissionCenterV1` 顶部布局结构

### 差异项

- title 改为客户侧 copy
- subtitle 改为客户侧 copy
- 右上按钮文案跟随 locale

---

## 6.2 Role Overview 表格

### 建议复用

- 表头结构
- 行操作位置
- 搜索与筛选区样式

### 差异项

- role list 改为客户侧 7 个模板角色
- `Special Control` 改为客户侧语义
- `Responsibility Description` 改为客户侧职责文案

---

## 6.3 Menu Permission Matrix 矩阵

### 建议复用

- 表格布局
- 分类折叠/展开
- 角色列横向排列

### 差异项

- 模块树改为客户侧模块基线
- 角色列改为客户侧角色基线
- 矩阵默认值改为客户侧 baseline

---

## 6.4 Actions & Data Scope

### 建议复用

- 左侧模块树
- 中部动作矩阵
- 右侧 scope 卡片区

### 差异项

- 默认动作集先维持 `View / Create / Edit / Delete / Approve`
- scope 文案改为客户侧英文源语言
- 默认范围逻辑改为企业内部协作逻辑

### 特别说明

`Purchaser` 在 `My Products` 下必须默认拥有：

- `View`
- `Create`
- `Edit`

---

## 6.5 User Exception Authorization

### 建议复用

- 顶部提示条结构
- 搜索、筛选、主按钮布局
- 空状态区

### 差异项

- 例外类型枚举改为客户侧业务语义
- 空状态、notice 改为客户侧 copy

---

## 6.6 Change Log

### 建议复用

- 时间筛选布局
- 搜索框布局
- 按钮布局
- 空状态布局

### 差异项

- 日志类型显示名跟随客户侧 copy
- 只展示客户侧权限中心相关日志

## 八、与现有 locale 体系的衔接

从现有代码看，客户侧已有 locale 状态可参考：

- [CustomerDashboard.tsx](/Users/luke/Documents/New%20project%202/innoshop_react20260221/src/components/CustomerDashboard.tsx)

### 建议接法

客户侧 Permission Center：

- 从客户侧主页接收 `locale`
- 使用客户侧 copy adapter 读取文案
- 不依赖我方侧语言切换状态

### 必须保证

- 客户侧 locale 变化时，Permission Center 立即刷新展示文案
- 我方侧后台 locale 不被影响

## 九、RTL 适配建议

### 适用语言

- `العربية`

### 建议处理方式

- 页面根节点加 `dir="rtl"`
- 对 code 列、枚举 code、英文标识使用 `dir="ltr"`
- 按钮顺序和表头对齐随 direction 自然翻转
- 不单独做一套 Arabic 页面

### 重点检查区域

- tabs
- 搜索框 icon 位置
- 表格列头对齐
- 右侧数据范围卡片
- 空状态图标和文案

---

## 十、复用与新增的建议结论

### 应优先复用

- 页面整体骨架
- 5 个 tab 的结构布局
- 表格与矩阵样式
- 顶部按钮流转
- 例外授权与日志页的空状态结构

### 应通过适配层替换

- 文案 copy
- 角色模板
- 模块树
- 矩阵基线
- 数据范围基线
- locale 驱动

### 不建议直接复用

- 我方侧角色字典
- 我方侧模块字典
- 我方侧区域型数据范围逻辑
- 我方侧平台级模块能力
