# 权限中心 V1 开发任务拆解清单

## 一、目标

把 **权限中心 V1** 从方案文档推进到可实施清单，供后续开发直接执行。

范围：
- 只针对我方侧
- 新建独立权限中心模块
- 不破坏旧图1、图2
- 不把逻辑继续堆到 `supabaseService.ts`

---

## 二、开发总原则

1. 新模块独立创建
2. 旧图1、图2先保留
3. 风格沿用现有系统
4. 布局参考大厂权限中心分层
5. 企业主数据中心继续负责“人和账号”
6. 权限中心只负责：
   - 角色
   - 模块
   - 动作
   - 数据范围
   - 例外授权
   - 变更日志

---

## 三、任务拆解总览

开发任务拆成 6 组：

1. 数据库
2. 基础字典初始化
3. 服务层
4. 前端页面
5. 联调
6. 验收

---

## 十、结论

权限中心 V1 的开发应该按：

`数据库`
→ `服务层`
→ `菜单主干`
→ `动作与范围`
→ `人员例外`
→ `日志与验收`

逐层推进。

这样既符合你当前“先新建，不破坏旧图1图2”的要求，也符合大厂常见的权限中心建设顺序。
