# 我方侧正式角色清单 V2（扩展版）

## 目标

用于统一我方侧内部角色口径，作为以下模块的共同基础：

- 企业主数据中心
- 权限中心
- 组织与行政
- 销售、采购、单证、财务、仓配等业务模块

本清单只覆盖**我方侧正式内部/外部协作登录角色**，不包含客户侧、供应商侧、第三方侧门户角色。

---

## 角色设计原则

1. 角色只表达“职责类型”，不表达具体人员
2. 区域是字段，不是角色
3. 具体人员不写进角色编码
4. 内部正式岗位与外部协作岗位分开
5. 管理角色、执行角色、协作角色分层清晰

---

## 一、系统与高管层

### 1. `Admin`
- 中文名：系统管理员
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 系统配置
  - 账号开通/停用
  - 权限配置
  - 菜单与动作授权
  - 审计与安全管理

### 2. `CEO`
- 中文名：首席执行官
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 经营总览
  - 战略决策
  - 核心审批
  - 全局业务监督

### 3. `CFO`
- 中文名：首席财务官
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 财务管理与管控
  - 收汇、结汇、付款控制
  - 财务合规
  - 财税决策

---

## 二、销售线角色

### 4. `Sales_Director`
- 中文名：销售总监
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 销售团队管理
  - 销售目标与预测
  - 重点客户管理
  - 报价/合同关键审批

### 5. `Regional_Manager`
- 中文名：区域主管
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：是
- 区域字段建议：
  - `NA`
  - `SA`
  - `EA`
  - `all`
- 主要职责：
  - 管理本区域业务员
  - 接收区域客户提交的 `ING`
  - 分配本区域业务员
  - 监督区域业务推进

### 6. `Sales_Rep`
- 中文名：业务员
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：是
- 主要职责：
  - 跟进 `ING -> QR -> QT -> SC`
  - 客户沟通
  - 合同、验货、付款、到港、收货、反馈跟进
  - 客户关系维护

### 7. `Sales_Assistant`
- 中文名：业务助理
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：建议是
- 主要职责：
  - 协助业务员整理询价、报价、合同资料
  - 跟进记录维护
  - 基础通知与客户回执催办
  - 支持区域销售团队日常执行

### 8. `Order_Coordinator`
- 中文名：跟单员
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：可选
- 主要职责：
  - 跟进 `SC -> PR -> CG -> 出运 -> 收货`
  - 节点催办
  - 单据齐套
  - 交期与交付协同
  - 跨部门订单推进

## 三、财务线角色

### 9. `Finance`
- 中文名：内部财务
- 类型：内部
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 收款
  - 付款
  - 财务单证
  - 内账管理
  - 财务包与归档

### 10. `External_Accountant`
- 中文名：代理记账财务
- 类型：外部协作登录角色
- 是否允许登录：是
- 是否需要区域字段：否
- 主要职责：
  - 获取代理记账所需财税资料
  - 对接税务局相关单证
  - 回填代理记账/报税处理结果
- 说明：
  - 该角色不是内部财务
  - 只应开放受限的财税资料范围
  - 不应拥有全经营数据权限

## 八、明确不再作为正式角色的写法

以下不建议继续作为正式角色：

- `Sales_Manager`
  - 已确认不保留
- `regional_manager_na`
- `regional_manager_sa`
- `regional_manager_emea`
  - 区域应作为字段，不应写进角色
- `sales_rep_maria`
- `sales_rep_ana`
- `sales_rep_emma`
  - 具体人员不能作为角色
- `system_admin`
  - 统一并入 `Admin`
- `marketing`
  - 若仅表示泛营销角色，统一并入 `Marketing_Ops`
