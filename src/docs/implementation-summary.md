### ✅ 阶段2：智能推荐算法（已完成）

#### **新增核心函数：**

#### 1. **getWorkloadLevel(workload)**
负载等级分析函数
```typescript
// 返回负载等级信息
{
  level: 'low' | 'medium' | 'high',
  color: string,      // 文字颜色
  bgColor: string,    // 背景颜色
  label: string,      // 等级标签
  icon: string        // 图标 (🟢/🟡/🔴)
}

// 分级规则：
- 0-3个询价单：🟢 空闲 (#22C55E)
- 4-6个询价单：🟡 适中 (#F59E0B)
- 7+个询价单：🔴 繁忙 (#EF4444)
```

#### 2. **getRecommendedSalesRep(region, mode)**
智能业务员推荐函数
```typescript
// 参数：
- customerRegion: 客户所在区域
- mode: 'manual' | 'least_load' | 'round_robin'

// 返回：
{
  recommended: string[],  // 推荐的业务员姓名列表
  all: Personnel[]        // 所有业务员信息（按负载排序）
}

// 模式说明：
- manual: 显示所有同区域业务员（按负载排序）
- least_load: 只推荐负载最低的业务员
- round_robin: 轮询分配（预留）
```

#### 3. **getRecommendedRecipientsGrouped(notifier)**
分组推荐函数（预留扩展）
```typescript
// 返回分组的推荐列表：
{
  priority: Personnel[],    // 优先推荐（业务员，按负载排序）
  management: Personnel[],  // 管理层
  others: Personnel[]       // 其他
}
```

#### 4. **增强原有getRecommendedRecipients()**
- 保持向后兼容
- 新增：客户询价时业务员推荐自动按负载排序
- 负载低的业务员优先出现在推荐列表中

## 🚀 如何使用

### **步骤1：打开泳道图**
1. 进入系统
2. 点击任意步骤的弹窗
3. 查看"📢 通知信息"区域

### **步骤2：选择触发人**
1. 点击"选择触发人"按钮
2. 选择一个客户（如：ABC Building Supplies 北美区）
3. 系统自动匹配同区域业务员

### **步骤3：查看被通知人推荐**
1. 点击"选择被通知人"按钮
2. 查看业务员列表：
   - 🟢 王建 (北美区) 3 ← 负载最低
   - 🟡 张伟 (北美区) 5
   - 🔴 李明 (北美区) 7 ← 负载最高

### **步骤4：快速分配（推荐）**
1. 点击"⚡ 快速分配"按钮
2. 系统自动选中负载最低的业务员
3. 完成！

### **步骤5（可选）：手动选择**
1. 也可以手动勾选多个业务员
2. 点击"全选"选中所有同区域业务员
3. 点击"清空"取消所有选择
