# 采购订单文档功能说明

## 📋 文档概述

采购订单文档（PurchaseOrderDocument.tsx）是福建高盛达富建材有限公司B2B外贸系统中的核心单证之一，用于向供应商发出正式采购订单。

**文档语言**：全中文
**文档格式**：A4标准（210mm × 297mm）
**主题色调**：橙色 #F96302

---

## ✅ 核心功能模块

### 1. 页眉区域
- **左侧**：公司Logo（高盛达富COSUN标识）
- **中间**：橙色横幅标题"采购订单"
- **右侧**：订单关键信息
  - 订单编号（橙色加粗）
  - 订单日期（中文格式）
  - 要求交期（橙色强调）

### 2. 双方信息区块
**采购方（买方）信息：**
- 中文公司名称
- 英文公司名称
- 地址
- 电话
- 邮箱
- 联系人（采购员）

**供应商信息：**
- 公司名称
- 供应商编码（可选）
- 地址
- 联系人
- 电话
- 邮箱

### 3. 🏦 **供应商银行收款信息**（独立突出显示）

这是采购订单的核心新功能，采用橙色主题边框独立显示：

**基础银行信息：**
- 开户银行（例：中国工商银行深圳宝安支行）
- 账户名称（例：Shenzhen Electronics Manufacturing Co., Ltd.）
- **银行账号**（橙色加大字体，核心信息）
- 收款币种（例：CNY/USD）

**国际汇款信息（可选）：**
- SWIFT代码（例：ICBKCNBJSZN）
- IBAN号码（欧洲地区）
- 路由号码（美国地区）
- 银行地址

**显示特点：**
- 淡橙色背景（bg-orange-50）
- 橙色双线边框
- 左侧橙色竖条标识
- 银行账号特大字号橙色显示
- 两列网格布局，合理利用空间

### 4. 采购明细表格
**表头字段（全中文）：**
- 序号
- 型号
- 图片
- 产品名称/规格
- 数量
- 单位
- 单价
- 金额

**表格特性：**
- 支持产品图片显示
- 支持逐项备注
- 底部显示"采购总金额"（橙色加粗）

### 5. 采购条款
**蓝色背景框显示：**
- 付款条款
- 交货条款
- 交货地址
- 质量标准
- 验收方式

### 6. 特殊要求（可选）
- 黄色背景显示
- 列表形式呈现

### 7. 备注（可选）
- 灰色背景
- 左侧橙色边框强调

### 8. 签章区域
**采购方（盖章）：**
- 公司中文名称
- 公司英文名称
- 虚线框：公司盖章处
- 授权代表签字栏
- 日期填写栏

**供应方（盖章）：**
- 供应商公司名称
- 供应商编码（如有）
- 虚线框：公司盖章处
- 授权代表签字栏
- 日期填写栏

### 9. 页脚说明
```
本采购订单一式两份，采购方和供应方各执一份，双方签章后生效。
请供应方签章确认后回传扫描件至采购方邮箱。
```

## 🎨 设计特点

### 视觉层次
```
Logo + 橙色标题横幅 + 订单信息
    ↓
采购方/供应方基础信息（灰色背景）
    ↓
橙色分隔线
    ↓
采购明细表格
    ↓
🔥 银行收款信息（橙色边框 - 特别醒目）← 位置优化
    ↓
采购条款（蓝色背景）
    ↓
特殊要求（黄色背景）/ 备注
    ↓
双方签章区域
```

### 色彩系统
- **主题色**：橙色 #F96302（标题、重点信息、银行账号）
- **辅助色**：
  - 灰色背景（信息区块）
  - 蓝色背景（采购条款）
  - 黄色背景（特殊要求）
  - 橙色浅背景（银行信息）

### 字体系统
- **中文字体**：Microsoft YaHei, SimHei
- **标准字号**：10pt
- **重要信息**：加大字号（银行账号18pt）

## 📊 数据接口

### PurchaseOrderData 接口结构

```typescript
export interface PurchaseOrderData {
  // 基本信息
  poNo: string;                    // 订单编号
  poDate: string;                  // 订单日期
  requiredDeliveryDate: string;    // 要求交期
  
  // 采购方信息
  buyer: {
    name: string;                  // 中文名称
    nameEn: string;                // 英文名称
    address: string;               // 中文地址
    addressEn: string;             // 英文地址
    tel: string;
    email: string;
    contactPerson: string;         // 采购员
  };
  
  // 供应商信息（含银行收款信息）
  supplier: {
    companyName: string;
    address: string;
    contactPerson: string;
    tel: string;
    email: string;
    supplierCode?: string;         // 供应商编码
    
    // 🏦 银行收款信息
    bankInfo?: {
      bankName: string;            // 开户银行
      accountName: string;         // 账户名称
      accountNumber: string;       // 银行账号 ⭐
      swiftCode?: string;          // SWIFT代码
      bankAddress?: string;        // 银行地址
      routingNumber?: string;      // 路由号码（美国）
      iban?: string;               // IBAN号码（欧洲）
      currency?: string;           // 收款币种
    };
  };
  
  // 产品清单
  products: Array<{
    no: number;
    modelNo?: string;
    imageUrl?: string;
    itemCode?: string;
    description: string;
    specification: string;
    quantity: number;
    unit: string;
    unitPrice: number;
    currency: string;
    amount: number;
    deliveryDate?: string;
    remarks?: string;
  }>;
  
  // 采购条款
  terms: {
    totalAmount: number;
    currency: string;
    paymentTerms: string;
    deliveryTerms: string;
    deliveryAddress: string;
    qualityStandard: string;
    inspectionMethod: string;
  };
  
  // 其他信息
  specialRequirements?: string[];
  remarks?: string;
  approval?: {
    requester: string;
    approver: string;
    approvalDate?: string;
  };
}
```

## 🔄 业务流程集成

### 在系统中的位置

```
销售合同签订
    ↓
业务员发起采购需求
    ↓
采购部门创建【采购订单】← 这里
    ↓
发送给供应商（含银行收款信息）
    ↓
供应商签章确认
    ↓
财务部门按银行信息付款
    ↓
供应商发货
    ↓
收货验收
```

### 关键使用场景

1. **采购部门**：创建采购订单，填写供应商银行信息
2. **财务部门**：根据订单上的银行信息进行付款
3. **供应商**：签章确认订单，回传扫描件
4. **审计部门**：审核采购订单的完整性和合规性

## ✨ 核心优势

### 1. 专业性
- 符合中国B2B采购合同标准格式
- 双语信息（中英文），便于国际供应商理解
- 法律效力完整（双方签章区域）

### 2. 实用性
- **银行收款信息醒目**：财务人员可直接参考付款
- 信息完整：从基本信息到付款条款一应俱全
- 易于打印：A4标准格式

### 3. 品牌一致性
- 橙色主题贯穿全文档
- COSUN品牌标识清晰
- 台湾大厂风格设计

### 4. 数据可追溯
- 订单编号
- 供应商编码
- 审批信息
- 关联项目清晰

---

## 📝 注意事项

1. **银行信息准确性**：银行账号等信息务必准确无误
2. **签章要求**：双方盖章后方可生效
3. **回传确认**：要求供应商签章后回传扫描件
4. **存档管理**：原件需妥善保管
