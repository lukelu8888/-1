## 角色定义

### 🎯 CEO（老板）- 最高决策者
- **职责**: 全局掌控、战略决策、风险控制
- **特点**: 所有数据可见、最终审批权、全模块访问
- **典型场景**: 查看公司整体业绩、审批大额合同、风险控制决策

### 💼 CFO（财务总监）- 财务负责人
- **职责**: 财务管理、现金流控制、成本分析、财务审批
- **特点**: 财务全权限、业务只读、能看成本利润
- **典型场景**: 财务分析、成本控制、付款审批、现金流管理

### 🎯 Sales Director（销售总监）- region='all'
- **职责**: 全球销售管理、销售团队管理、销售策略制定
- **特点**: 所有区域销售数据、能看毛利润、销售审批权
- **典型场景**: 管理所有区域销售团队、审批报价和合同、查看销售业绩

### 👔 Regional Manager（区域主管）- region='NA'/'SA'/'EMEA'
- **职责**: 区域销售管理、区域团队管理、区域客户开发
- **特点**: 仅本区域数据、能看区域毛利、区域审批权
- **典型场景**: 管理北美/南美/欧非销售团队、审批区域报价和合同

### 👨‍💼 Sales Rep（业务员）- 一线销售
- **职责**: 客户开发、询价跟进、报价提交、订单跟进
- **特点**: 只能看自己的数据、不能看利润、提交需审批
- **典型场景**: 开发客户、跟进询价、提交报价（需主管审批）

### 🛒 Procurement（采购专员）- 采购负责人
- **职责**: 供应商管理、采购订单、成本控制、付款跟进
- **特点**: 后台供应链全权限、前台只读、能看成本
- **典型场景**: 管理供应商、下采购订单、控制采购成本

### 💰 Finance（财务专员）- 财务执行
- **职责**: 收款登记、付款登记、账务核对、财务报表
- **特点**: 财务执行权限、业务只读、不能看利润率
- **典型场景**: 登记收款、登记付款、核对账务、生成报表

### 🔧 Admin（系统管理员）- 技术支持
- **职责**: 系统配置、用户管理、数据备份、技术支持
- **特点**: 系统管理全权限、业务数据只读
- **典型场景**: 用户管理、系统配置、数据备份、技术支持

**💡 设计理念：**
- **销售部门（销售总监/区域主管）**：可以查看应收账款和收款管理，用于督促回款、评估业绩，但不能查看应付和付款（前后台分离）
  - **销售总监**：可以查看所有区域的应收和回款情况（查看权限，不能操作）
  - **区域主管**：可以查看本区域的应收和回款情况（查看权限，不能操作）
  - **业务员**：只能查看自己客户的应收账款，不能查看收款管理
- **采购部门**：可以查看应付账款和付款管理，用于跟进付款进度，但不能查看应收和收款（职责分离）
- **财务部门（CFO/财务）**：可以访问全部财务模块（应收+应付+收款+付款），统筹管理资金流转

- **特殊权限**: 可以查看所有区域毛利润

- **范围**: 仅北美区域
- **客户**: 仅北美区域客户
- **订单**: 仅北美区域订单
- **数据过滤**: `region = 'NA'`
- **特殊权限**: 可以查看北美区域毛利润

#### 南美区域主管（region='SA'）
- **范围**: 仅南美区域
- **客户**: 仅南美区域客户

- **数据过滤**: `region = 'EMEA'`

## 审批权限说明

### 🔄 B2B外贸审批流程

#### 询价 → 报价审批流程
```
业务员提交报价
    ↓
区域主管审批（小额：< $50,000）
    ↓
销售总监审批（大额：≥ $50,000）
    ↓
CEO最终审批（超大额：≥ $200,000）
```

#### 报价 → 合同审批流程
```
业务员提交合同
    ↓
区域主管审批（小额合同）
    ↓
销售总监审批（大额合同）
    ↓
CEO最终审批（超大额合同）
```

#### 采购 → 付款审批流程
```
采购提交付款申请
    ↓
CFO审批（财务审核）
    ↓
CEO最终审批（大额付款）
```

#### 销售 → 收款审批流程
```
财务登记收款
    ↓
CFO审批（财务确认）
    ↓
CEO知晓（大额收款）
```

## 动态权限扩展机制

### 🎯 销售总监（region='all'）动态扩展

销售总监的权限通过`hasPermission`函数动态扩展：

```typescript
if (user.role === 'Sales_Manager' && user.region === 'all') {
  // 1. 数据范围扩展
  if (permission === 'view:all_data') return true;
  
  // 2. 财务数据扩展
  if (permission === 'view:profit_margin') return true;
  
  // 3. 供应商数据扩展
  if (permission === 'view:supplier_data') return true;
  
  // 4. 模块访问扩展
  if (permission === 'access:service_provider') return true;
  
  // 5. 审批权限扩展
  if (permission === 'approve:quotation_final') return true;
  if (permission === 'approve:contract_final') return true;
}
```

### 🔧 区域主管（region='NA'/'SA'/'EMEA'）

区域主管使用基础权限配置，通过数据过滤层实现region级别的数据隔离：

```typescript
// 数据查询时自动过滤
const filteredData = allData.filter(item => 
  item.region === currentUser.region
);
```

## 📝 权限配置最佳实践

### 1. 最小权限原则
- 每个角色只拥有完成其工作所需的最小权限
- 业务员只能看自己的数据，不能看利润
- 区域主管只能看自己区域的数据

### 2. 职责分离原则
- 前台（销售）和后台（采购/财务）分离
- 业务和财务分离
- 采购不能访问客户信息（保护客户隐私）
- 销售不能访问采购成本（保护供应商信息）

### 3. 数据安全原则
- 利润率和成本明细仅限CEO/CFO查看
- 业务员不能看到公司利润率（防止泄露）
- 区域主管不能看到详细成本（防止泄露供应商价格）

### 4. 审批流程原则
- 小额业务区域主管审批
- 大额业务销售总监审批
- 超大额业务CEO最终审批
- 财务付款CFO审批

### 5. 灵活扩展原则
- 通过region字段区分销售总监和区域主管
- 通过动态权限函数扩展权限
- 不需要创建新角色，保持角色体系简洁

   - 不应该看到：供应商、采购订单、成本明细

## 🎯 总结

本权限配置系统完全基于**B2B外贸实战经验**设计：

✅ **前后台分离** - 销售和采购各司其职，数据隔离  
✅ **业财分离** - 业务和财务职责清晰，防止利益冲突  
✅ **分级审批** - 根据金额大小分级审批，提高效率  
✅ **数据安全** - 敏感数据（利润、成本）严格控制访问  
✅ **灵活扩展** - 通过region字段和动态权限实现灵活配置  
✅ **易于维护** - 角色清晰、权限明确、便于管理
