# 📐 步骤1 UI设计方案：触发人 + 被通知人

## 🎯 设计目标

### **用户问题：**
> 步骤1的触发人有3家公司，被通知人也可以选很多人，应该怎么设计？

### **核心挑战：**
1. **触发人**：3个客户（ABC、Brasil、Europa）
2. **被通知人**：9个业务员 + 3个主管 + 1个总监 + 其他
3. **关系复杂**：每个客户对应不同的业务员
4. **场景多样**：测试环境 vs 生产环境

---

## 🛠️ 技术实现

### **1. 触发人选择逻辑**

```typescript
interface NotificationConfig {
  // null = 任意客户（动态检测）
  // ['ABC'] = 仅ABC客户
  // ['ABC', 'Brasil'] = ABC或Brasil客户
  selectedCustomers: string[] | null;
  
  // 运行模式
  mode: 'specific' | 'dynamic';
}

// 示例1：指定客户
{
  selectedCustomers: ['ABC Building Supplies'],
  mode: 'specific'
}

// 示例2：动态检测
{
  selectedCustomers: null,
  mode: 'dynamic'
}
```

### **2. 被通知人推荐逻辑**

```typescript
function getRecommendedRecipients(
  selectedCustomers: string[] | null
): Personnel[] {
  
  // 情况1：未选择客户（动态模式）
  if (!selectedCustomers || selectedCustomers.length === 0) {
    return []; // 运行时动态路由，不预设
  }
  
  // 情况2：选择了具体客户
  const recommended: Personnel[] = [];
  
  selectedCustomers.forEach(customerName => {
    // 查询该客户的对接业务员
    const customer = findPersonnelByName(customerName);
    if (!customer || !customer.region) return;
    
    // 智能路由：查询或分配
    const salesRep = routeToSalesRep(customerName, customer.region);
    
    if (salesRep) {
      recommended.push(salesRep);
    }
  });
  
  return recommended;
}
```

### **3. UI交互流程**

```typescript
const [selectedCustomers, setSelectedCustomers] = useState<string[]>([]);
const [recommendedRecipients, setRecommendedRecipients] = useState<Personnel[]>([]);

// 当触发人改变时，自动更新推荐的被通知人
useEffect(() => {
  const recommended = getRecommendedRecipients(selectedCustomers);
  setRecommendedRecipients(recommended);
}, [selectedCustomers]);

// 渲染被通知人列表
function renderRecipients() {
  if (selectedCustomers.length === 0) {
    return <div>🤖 智能路由（运行时自动匹配）</div>;
  }
  
  return (
    <div>
      <h4>🌟 推荐的业务员</h4>
      {recommendedRecipients.map(person => (
        <Checkbox
          key={person.name}
          defaultChecked={true}
          label={`${person.displayName} ${getWorkloadBadge(person.workload)}`}
        />
      ))}
      
      <h4>👥 其他可选</h4>
      {/* ...其他人员 */}
    </div>
  );
}
```

## 📐 详细界面设计规范

### **组件结构：**

```tsx
<StepNotificationConfig stepId={1}>
  
  {/* 触发人部分 */}
  <TriggerPersonSelector
    title="触发人"
    options={[
      { id: 'ABC', label: 'ABC Building Supplies', region: 'north_america' },
      { id: 'Brasil', label: 'Brasil Construction Co.', region: 'south_america' },
      { id: 'Europa', label: 'Europa Trading GmbH', region: 'europe_africa' },
      { id: 'any', label: '🤖 任意客户（动态检测）', isSpecial: true }
    ]}
    value={selectedCustomers}
    onChange={setSelectedCustomers}
  />
  
  {/* 被通知人部分 */}
  <RecipientSelector
    title="被通知人"
    mode={selectedCustomers.length > 0 ? 'smart-recommend' : 'dynamic'}
    recommendedPersonnel={recommendedRecipients}
    allPersonnel={getAllPersonnelWithSuppliers()}
    onRecipientsChange={setSelectedRecipients}
  />
  
  {/* 映射关系显示 */}
  {selectedCustomers.length > 0 && (
    <MappingInfoPanel
      customers={selectedCustomers}
      mappings={selectedCustomers.map(c => ({
        customer: c,
        salesRep: routeToSalesRep(c, getCustomerRegion(c)),
        assignedDate: '...',
        assignedBy: 'auto'
      }))}
    />
  )}
  
</StepNotificationConfig>
```
