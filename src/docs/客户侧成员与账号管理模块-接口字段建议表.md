# 客户侧成员与账号管理模块接口字段建议表

## 一、目标

本文用于定义客户侧 `People & Accounts Center` 的接口输入输出建议字段，方便前后端对齐。

适用范围：

- `People Directory`
- `Account Access`
- `Role Permissions`

---

## 二、接口清单建议

建议至少提供：

1. `GET /customer/people-accounts/members`
2. `GET /customer/people-accounts/members/:id`
3. `POST /customer/people-accounts/members/invite`
4. `POST /customer/people-accounts/members/:id/resend-invite`
5. `PATCH /customer/people-accounts/members/:id/account-status`
6. `PATCH /customer/people-accounts/members/:id/role`
7. `GET /customer/people-accounts/roles`
8. `GET /customer/people-accounts/account-access`

---

## 三、基础字段建议

## 3.1 Member

| Field | Type | Example | Notes |
|---|---|---|---|
| `memberId` | `string` | `mem_001` | 成员主键 |
| `employeeNo` | `string` | `CS-001` | 工号，可为空 |
| `fullName` | `string` | `John Smith` | 显示名 |
| `department` | `string` | `Procurement` | 部门 |
| `title` | `string` | `Purchaser` | 职位 |
| `businessEmail` | `string` | `john@client.com` | 业务邮箱 |
| `loginEmail` | `string` | `john.login@client.com` | 登录邮箱 |
| `roleCode` | `string` | `Purchaser` | 稳定 code |
| `roleName` | `string` | `Purchaser` | locale 显示名 |
| `memberStatus` | `string` | `active` | 成员状态 |
| `canLogin` | `boolean` | `true` | 是否允许登录 |
| `lastLoginAt` | `string \| null` | `2026-04-05T09:00:00Z` | 最近登录 |

## 3.2 Account Status

| Field | Type | Example | Notes |
|---|---|---|---|
| `accountStatusCode` | `string` | `invited` | 稳定 code |
| `accountStatusName` | `string` | `Invited` | locale 显示名 |
| `inviteSentAt` | `string \| null` | `2026-04-05T08:00:00Z` | 邀请发送时间 |
| `inviteExpiresAt` | `string \| null` | `2026-04-12T08:00:00Z` | 邀请过期时间 |
| `activatedAt` | `string \| null` | `2026-04-06T10:30:00Z` | 激活时间 |

## 五、Invite / Account Access 接口

## 5.1 邀请成员

### Request

`POST /customer/people-accounts/members/invite`

```json
{
  "fullName": "John Smith",
  "title": "Purchaser",
  "businessEmail": "john@client.com",
  "loginEmail": "john.login@client.com",
  "roleCode": "Purchaser"
}
```

### Response

```json
{
  "success": true,
  "data": {
    "memberId": "mem_001",
    "inviteUrl": "https://example.com/invite/abc123",
    "accountStatusCode": "invited",
    "accountStatusName": "Invited",
    "inviteSentAt": "2026-04-05T08:00:00Z"
  }
}
```

## 5.2 重发邀请

### Request

`POST /customer/people-accounts/members/mem_001/resend-invite`

### Response

```json
{
  "success": true,
  "data": {
    "memberId": "mem_001",
    "inviteUrl": "https://example.com/invite/xyz789",
    "inviteSentAt": "2026-04-06T09:00:00Z"
  }
}
```

## 5.3 更新账号状态

### Request

`PATCH /customer/people-accounts/members/mem_001/account-status`

```json
{
  "accountStatusCode": "suspended"
}
```

### 允许值建议

- `active`
- `invited`
- `suspended`
- `locked`
- `pending_activation`

## 六、Role Permissions 接口

## 6.1 获取角色列表

### Request

`GET /customer/people-accounts/roles?locale=en`

### Response

```json
{
  "success": true,
  "data": {
    "roles": [
      { "roleCode": "Company_Owner", "roleName": "Company Owner" },
      { "roleCode": "Company_Admin", "roleName": "Company Admin" },
      { "roleCode": "Purchase_Manager", "roleName": "Purchase Manager" },
      { "roleCode": "Purchaser", "roleName": "Purchaser" }
    ]
  }
}
```

## 6.2 更新成员角色

### Request

`PATCH /customer/people-accounts/members/mem_001/role`

```json
{
  "roleCode": "Purchase_Manager"
}
```

### Response

```json
{
  "success": true,
  "data": {
    "memberId": "mem_001",
    "roleCode": "Purchase_Manager",
    "roleName": "Purchase Manager"
  }
}
```

### 关键要求

- 角色来源必须与 `Permission Center` 角色字典一致
- 普通业务角色不能通过接口把自己提升为管理员

## 八、locale 字段建议

建议所有读取接口支持：

| Field | Type | Example | Notes |
|---|---|---|---|
| `locale` | `string` | `en` | `en` / `es` / `pt` / `ar` |

### 规则

- 仅客户侧门户接口接收客户 locale
- 我方侧后台不跟随客户 locale

---

## 九、鉴权规则建议

所有写接口必须在后端校验：

- 只有 `Company_Owner` 和 `Company_Admin` 可以修改成员和账号
- 普通业务角色只能读自己被允许的数据
- 普通角色不能通过接口修改管理员角色

---

## 十、最终建议

这套接口的关键不是字段多少，而是三点：

1. 成员与账号信息要分清
2. 角色分配要和 Permission Center 角色字典打通
3. 管理员边界必须在后端写死
