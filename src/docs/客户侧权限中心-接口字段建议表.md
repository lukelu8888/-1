# 客户侧权限中心接口字段建议表

## 一、目标

本文用于定义客户侧 Permission Center 的接口输入输出建议字段，方便前后端对齐。

原则：

- 字段命名尽量稳定
- 与现有权限中心数据结构保持可复用性
- 客户侧与我方侧可共用底层 RBAC 模型
- 客户侧角色模板、模块树与 scope 语义单独控制

---

## 二、接口清单建议

建议客户侧至少提供以下接口：

1. `GET /customer/permission-center/roles`
2. `GET /customer/permission-center/menu-matrix`
3. `POST /customer/permission-center/menu-matrix`
4. `GET /customer/permission-center/module-actions`
5. `POST /customer/permission-center/module-actions`
6. `GET /customer/permission-center/data-scopes`
7. `POST /customer/permission-center/data-scopes`
8. `GET /customer/permission-center/exceptions`
9. `POST /customer/permission-center/exceptions`
10. `PATCH /customer/permission-center/exceptions/:id`
11. `POST /customer/permission-center/exceptions/:id/revoke`
12. `GET /customer/permission-center/change-logs`
13. `POST /customer/permission-center/publish`

---

## 三、公共字段建议

### 3.1 Role

| Field | Type | Example | Notes |
|---|---|---|---|
| `roleCode` | `string` | `Purchaser` | 不翻译，稳定 code |
| `roleName` | `string` | `Purchaser` | 当前 locale 下显示名 |
| `category` | `string` | `Business` | 当前 locale 下显示值或标准 enum |
| `specialControl` | `string` | `Assigned or Self Data` | 展示文案 |
| `responsibilityDescription` | `string` | `Maintain My Products, create inquiries, place orders...` | 展示文案 |
| `isSystemTemplate` | `boolean` | `true` | 是否系统默认模板 |
| `isEditable` | `boolean` | `true` | 前端控制是否允许编辑 |
| `sortOrder` | `number` | `4` | 表格排序 |

### 3.2 Module

| Field | Type | Example | Notes |
|---|---|---|---|
| `moduleCode` | `string` | `my-products` | 稳定 code |
| `moduleName` | `string` | `My Products` | 当前 locale 下显示名 |
| `moduleGroupCode` | `string` | `business` | 分组 code |
| `moduleGroupName` | `string` | `Business Modules` | 分组显示名 |
| `parentModuleCode` | `string \| null` | `null` | 模块树需要时使用 |
| `sortOrder` | `number` | `10` | 排序 |
| `isVisibleInMatrix` | `boolean` | `true` | 是否展示在矩阵中 |

## 四、Role Overview 接口

## 4.1 获取角色列表

### Request

`GET /customer/permission-center/roles?locale=en&category=all&keyword=purch`

### Response

```json
{
  "success": true,
  "data": {
    "roles": [
      {
        "roleCode": "Purchaser",
        "roleName": "Purchaser",
        "category": "Business",
        "specialControl": "Assigned or Self Data",
        "responsibilityDescription": "Maintain My Products, create inquiries, place orders, and follow up purchasing tasks.",
        "isSystemTemplate": true,
        "isEditable": true,
        "sortOrder": 4
      }
    ]
  }
}
```

### 字段要求

- `roleName` 返回当前 locale 文案
- `roleCode` 固定不翻译
- `responsibilityDescription` 必须包含 `My Products` 对于 `Purchaser`

## 五、Menu Permission Matrix 接口

## 5.1 获取菜单矩阵

### Request

`GET /customer/permission-center/menu-matrix?locale=en`

### Response

```json
{
  "success": true,
  "data": {
    "roles": [
      { "roleCode": "Company_Owner", "roleName": "Company Owner" },
      { "roleCode": "Purchaser", "roleName": "Purchaser" }
    ],
    "modules": [
      {
        "moduleGroupCode": "business",
        "moduleGroupName": "Business Modules",
        "items": [
          {
            "moduleCode": "my-products",
            "moduleName": "My Products"
          },
          {
            "moduleCode": "inquiry-center",
            "moduleName": "Inquiry Center"
          }
        ]
      }
    ],
    "matrix": {
      "Company_Owner": {
        "my-products": true,
        "inquiry-center": true
      },
      "Purchaser": {
        "my-products": true,
        "inquiry-center": true
      }
    }
  }
}
```

### 保存菜单矩阵 Request

`POST /customer/permission-center/menu-matrix`

```json
{
  "changes": [
    {
      "roleCode": "Purchaser",
      "moduleCode": "my-products",
      "visible": true
    }
  ]
}
```

### 验证要求

- `Purchaser` + `my-products` 初始值必须为 `true`

## 十、Publish 接口

## 10.1 发布权限变更

### Request

`POST /customer/permission-center/publish`

```json
{
  "changeSummary": {
    "roleChanges": 0,
    "menuMatrixChanges": 4,
    "actionChanges": 3,
    "scopeChanges": 2,
    "exceptionChanges": 1
  }
}
```

### Response

```json
{
  "success": true,
  "message": "Permission changes have been published successfully.",
  "data": {
    "publishedAt": "2026-04-05T12:00:00Z",
    "publishedBy": "Mary Johnson"
  }
}
```

### 验证要求

- 发布动作必须落 `change_logs`
- 发布后再次获取矩阵应返回最新值

## 十一、locale 字段建议

建议所有读取接口支持：

| Field | Type | Example | Notes |
|---|---|---|---|
| `locale` | `string` | `en` | `en` / `es` / `pt` / `ar` |

### 规则

- 仅客户侧接口接受客户门户 locale
- 我方侧接口不必跟随客户侧 locale

---

## 十二、租户边界字段建议

建议所有客户侧接口由后端从登录态解析租户，不允许前端自由传入任意租户 ID。

如确需返回租户相关字段，可在响应中附带：

| Field | Type | Example | Notes |
|---|---|---|---|
| `tenantId` | `string` | `cust_org_001` | 只回显，不建议由前端主导传入 |
| `portalType` | `string` | `customer` | 用于区分端类型 |

---

## 十三、最终建议

### 字段命名原则

- code 字段稳定，不翻译
- name 字段可本地化
- type、scope、changeType 优先使用 `xxxCode + xxxName` 双字段

### 前后端对齐原则

- 矩阵类接口尽量返回结构化数据
- 页面展示所需文案尽量返回 name，但 code 仍保留
- 客户侧权限模型与我方侧结构可兼容，基线数据必须分离
