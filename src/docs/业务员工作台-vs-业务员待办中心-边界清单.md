# 业务员工作台 vs 业务员待办中心 边界清单

## 目的

这份清单用于明确区分：

- 业务员工作台
- 业务员待办中心

避免两者在产品定位、页面布局、数据内容、动作入口上再次混淆。

本清单是后续设计、开发、评审的硬约束。

---

## 一、核心定位

### 1. 业务员工作台

业务员工作台是：

- 经营驾驶舱
- 客户经营分析台
- 销售过程总览台

它回答的问题是：

- 我的客户结构怎么样
- 我的潜客、现有客户、重点客户分布如何
- 我的转化率、成交额、预计佣金怎样
- 哪类客户值得优先投入
- 本月整体经营情况如何

### 2. 业务员待办中心

业务员待办中心是：

- 执行处理台
- 客户跟进动作台
- 节点催办与回写台

它回答的问题是：

- 我今天先联系谁
- 哪张单卡住了
- 当前卡点是什么
- 下一步动作是什么
- 我处理完后状态怎么回写

---

## 二、业务顺序基础

业务员待办中心必须建立在正式主链之上：

`ING -> QR -> XJ -> BJ -> QT -> SC -> PR -> CG -> 验货/付款/出运 -> 到港/收货 -> 反馈/结案`

其中业务员真正重点负责的链路是：

`ING -> QT -> SC -> 客户验货方式 -> 客户付款 -> 到港/清关/收货 -> 客户反馈`

---

## 三、业务员工作台应承载的内容

以下内容只允许出现在业务员工作台，不应进入待办中心：

### 1. 经营分析

- 客户总数
- 潜客数
- 成交数
- 转化率
- 预计佣金
- 销售收入
- 月度/季度趋势

### 2. 客户结构与分层

- 客户类型分布
- ABC 分级
- BANT 总览
- 区域分布
- 行业分布
- 来源渠道分布
- 现有客户 vs 潜在客户结构

### 3. 长周期计划与策略

- 客户开发执行手册
- 客户开发阶段漏斗
- 长周期行动清单
- 客户来源分析
- ROI 分析
- 经营差距与目标完成率

### 4. 管理视角信息

- 团队/区域总体盘子
- 目标达成率
- 本月收入结构
- 重点客户经营总览

---

## 四、业务员待办中心应承载的内容

以下内容只允许出现在业务员待办中心，不应被工作台替代：

### 1. 今日执行动作

- 今日必须联系客户
- 已超期未联系客户
- 今日应催付款客户
- 今日应确认验货方式客户
- 今日应确认收货客户
- 今日应推动反馈客户

### 2. 节点型业务待办

- 新 `ING` 待响应
- `ING` 待澄清
- `QR` 待成本回传
- `QT` 待发送
- `QT` 待客户反馈
- `SC` 待客户签署
- `SC` 待客户定金
- 待确认验货方式
- 待安排第三方验货
- 待安排第三方监装
- 待客户付款
- 待客户确认海运费/船期
- 待到港确认
- 待清关资料
- 待收货回执
- 待客户反馈
- 异常/客诉待协调

### 3. 跟进动作

- 已联系客户
- 已发 `QT`
- 已催合同
- 已催定金
- 已发验货报告
- 已催付款
- 已通知到港
- 已确认收货
- 约下次跟进
- 记录客户回复

### 4. 跟进节奏

- 最近联系时间
- 下次跟进时间
- 超期天数
- 今天是否已完成
- 最近客户回复

---

## 五、页面结构边界

### 1. 业务员工作台页面结构

工作台适合：

- 指标卡
- 图表
- 分布图
- 客户分层面板
- 漏斗
- 趋势分析
- 经营摘要

工作台不适合：

- 大量“待处理按钮”
- 逐单逐客的催办清单
- 高频行内动作

### 2. 业务员待办中心页面结构

待办中心适合：

- 紧凑工作条
- 今日必须处理摘要
- 按客户/单据分组的执行清单
- 行内动作按钮
- 最近客户回复
- 跟进记录
- 时间线抽屉

待办中心不适合：

- 客户类型业务指标大盘
- 来源渠道分析
- ABC/BANT 总览
- 经营趋势图
- ROI/转化率图表

---

## 六、数据组织方式边界

### 1. 工作台的数据组织方式

工作台按：

- 客户池
- 客户类型
- 客户分层
- 区域
- 阶段
- 渠道
- 经营指标

来组织。

### 2. 待办中心的数据组织方式

待办中心按：

- 客户
- 当前业务单
- 当前节点
- 当前卡点
- 下一步动作
- 最近联系/下次跟进

来组织。

不得把待办中心做成“经营分析页”，也不得把工作台做成“任务清单页”。

---

## 七、动作回写原则

业务员待办中心里的动作，必须能回写真实业务链。

例如：

- 已联系客户
- 已发 `QT`
- 已催合同
- 已催定金
- 已发报告
- 已催付款
- 已通知到港
- 已确认收货
- 已记录反馈

这些动作至少要回写：

- 跟进记录
- 最近联系时间
- 下次跟进时间
- 对应业务节点的最新状态或协同状态

工作台不承担这类高频动作回写。

---

## 八、禁止混放清单

以下内容禁止同时在两个模块重复出现：

### 禁止放进待办中心

- 客户类型业务指标
- 客户来源分析
- 社媒渠道分析
- BANT 总览
- ABC 分级总览
- 客户结构分布
- 月度经营趋势
- ROI 分析
- 预计佣金总盘
- 长周期客户开发地图

### 禁止放进工作台

- 今日必须联系谁
- 哪张 `QT / SC / Shipment` 卡住
- 超期未联系清单
- 最近客户回复待处理
- 行内催办按钮
- 跟进记录抽屉
- 逐单逐客的处理动作

---

## 九、产品命名原则

### 工作台命名

允许：

- 业务员工作台
- 客户经营工作台
- 销售工作台

### 待办中心命名

允许：

- 业务员待办中心
- 客户跟进中心
- 跟进处理中心

### 禁止命名

- 用“前段/后段”命名产品页面
- 用“通用待办模板”式名称误导角色
- 把工作台和待办中心命名成近似概念

---

## 十、最终判定规则

如果一个页面主要回答：

- “我的盘子怎么样”
- “我的客户结构怎么样”
- “本月经营情况怎么样”

那么它属于：

- **工作台**

如果一个页面主要回答：

- “我现在先联系谁”
- “哪张单卡住了”
- “下一步我该做什么”

那么它属于：

- **待办中心**

---

## 十一、后续开发约束

后续任何涉及业务员工作台或业务员待办中心的开发，必须先确认：

1. 功能展示在哪里
2. 属于工作台还是待办中心
3. 面向哪个角色
4. 页面是分析型还是执行型
5. 是否与现有模块职责冲突

未完成以上确认，不得直接开发页面。
