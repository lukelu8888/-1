# 🌐 通知内容与字段状态语言显示规则 - 测试指南

## 📋 语言显示规则说明

### 规则概述
系统根据**发送者角色**、**接收者角色**和**当前查看者角色**智能决定显示中文还是英文。

### 三大核心规则

#### 1️⃣ 客户发出的通知 → 接收人（其他角色）看英文
- **场景**：客户发送询价单、确认信息等
- **示例**：
  - 步骤1：客户提交询价 → 业务员查看时看到英文通知
  - 通知内容：`Inquiry notification sent to Sales Rep Zhang Wei`

#### 2️⃣ 其他角色发给客户的通知 → 客户看英文
- **场景**：业务员、财务、物流等通知客户
- **示例**：
  - 步骤50：业务员通知客户完货 → 客户查看时看到英文通知
  - 通知内容：`Customer has been notified that goods are ready with inspection report attached.`

#### 3️⃣ 非客户之间互发的通知 → 都看中文
- **场景**：内部员工之间的通知
- **示例**：
  - 步骤3：业务员通知采购员 → 采购员查看时看到中文通知
  - 通知内容：`已通知采购员李明处理询价请求`

## 🧪 测试步骤

### 1. 打开V5全流程演示弹窗

1. 访问Admin Portal
2. 找到"订单管理中心"
3. 点击任意订单查看详情
4. 打开"🎬 全流程演示"弹窗

### 2. 测试视角切换功能

弹窗标题栏右侧有一个**视角切换器**下拉框，默认显示"业务员"。

#### 测试场景A：切换到"客户"视角

1. 将视角切换为"**客户**"
2. 点击步骤1（客户提交询价）
3. 查看"通知信息"区域
4. **预期结果**：
   - 标题显示：`Notification`（英文）
   - 字段标签：`Notifier` / `Recipients` / `Notification Content`
   - 通知内容：`Inquiry notification sent to Sales Rep Zhang Wei`（英文）
   - 字段标签：`Form Fields`（英文）
   - 所有字段名：`Inquiry Status`, `Product Type`, `Target Quantity`等（英文）

#### 测试场景B：切换到"业务员"视角

1. 将视角切换为"**业务员**"
2. 点击步骤1（客户提交询价）
3. 查看"通知信息"区域
4. **预期结果**：
   - 标题显示：`通知信息`（中文）
   - 字段标签：`通知人` / `被通知人` / `通知内容`
   - 通知内容：`已向业务员张伟发送询盘通知`（中文）
   - 字段标签：`表单字段`（中文）
   - 所有字段名：`询盘状态`、`产品类型`、`目标数量`等（中文）

#### 测试场景C：内部通知（业务员→采购员）

1. 将视角切换为"**采购员**"
2. 点击步骤3（业务员分配采购）
3. 查看"通知信息"区域
4. **预期结果**：
   - 通知内容：`已通知采购员李明处理询价请求`（中文）
   - 因为是内部员工之间的通知，采购员看到的是中文

5. 将视角切换为"**客户**"
6. 再次查看步骤3
7. **预期结果**：
   - 通知内容：`Procurement Officer Li Ming has been notified to process the inquiry request.`（英文）
   - 虽然这是内部通知，但客户视角下所有内容都显示英文

#### 测试场景D：业务员通知客户

1. 将视角切换为"**客户**"
2. 点击步骤50（通知客户完货）
3. 查看"通知信息"区域
4. **预期结果**：
   - 通知内容：`Customer has been notified that goods are ready with inspection report attached.`（英文）
   - 被通知人：`客户John Smith`
   - 客户看到的是英文通知

5. 将视角切换为"**业务员**"
6. 再次查看步骤50
7. **预期结果**：
   - 通知内容：`已通知客户完货，附验货报告`（中文）
   - 业务员查看自己发出的通知时，看到的是中文

## 🐛 已知问题

### 问题1：部分步骤缺少英文翻译
- **影响**：切换到客户视角时，部分通知内容没有英文版本，会fallback到中文
- **解决方案**：逐步为所有102个步骤添加英文翻译

### 问题2：人名暂不翻译
- **现状**：通知人和被通知人的姓名暂时保持原文（如"张伟"、"John Smith"）
- **原因**：人名翻译比较复杂，需要维护人员名录的双语版本
- **计划**：后续可以在personnelByRole数据中添加英文名字段

## 💡 扩展应用

### 在真实业务系统中应用

这套语言规则不仅适用于V5演示弹窗，还可以应用到：

1. **订单管理中心**的订单详情页
2. **客户Portal**的订单查看页面
3. **供应商Portal**的订单信息页面
4. **邮件通知**系统（根据收件人角色自动选择语言）
5. **PDF文档**生成（根据目标受众自动选择语言）

### 实现方式

使用 `/lib/notification-language.ts` 中的工具函数：

```typescript
import { getNotificationLanguage, selectLanguageText } from '@/lib/notification-language';

// 判断应该显示的语言
const language = getNotificationLanguage(
  viewerRole,      // 当前查看者角色
  notifierRole,    // 通知发送者角色
  recipientRole    // 通知接收者角色（可选）
);

// 选择对应语言的文本
const text = selectLanguageText(
  language,
  '已完成发货',    // 中文文本
  'Shipment completed'  // 英文文本
);
```
