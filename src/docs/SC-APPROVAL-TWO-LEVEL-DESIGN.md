## 🎯 业务需求

### **当前问题：**
- ❌ 步骤12：销售总监直接审批（不符合实际）
- ❌ 没有体现区域主管的审批权限
- ❌ 没有体现金额分级审批逻辑

### **实际业务流程：**

```
销售合同审批流程（两级审批）
━━━━━━━━━━━━━━━━━━━━━━━━━━━

步骤11：业务员提交销售合同（SC）
   ↓
步骤12：区域主管审批（一审，必须）
   ├─ 审批内容：合同条款、价格、交期
   ├─ 审批权限：<$20,000可直接批准
   └─ 审批结果：
       ├─ 通过 → 判断金额
       │    ├─ 金额 < $20,000 → 步骤14（完成审批）
       │    └─ 金额 ≥ $20,000 → 步骤13（需总监审批）
       └─ 拒绝 → 退回业务员修改
   ↓
步骤13：销售总监审批（二审，条件性）
   ├─ 触发条件：合同金额 ≥ $20,000
   ├─ 审批内容：大额合同风险评估
   ├─ 审批人：王强（销售总监）
   └─ 审批结果：
       ├─ 通过 → 步骤14（完成审批）
       └─ 拒绝 → 退回业务员修改
   ↓
步骤14：采购员拆单（原步骤13）
```

---

## 💡 为什么要拆分成两个步骤？

### **原因1：符合真实审批流程** ✅

```
真实企业审批：
┌─────────────────────────────────────────┐
│ 一级审批：区域主管（必须）               │
│ ├─ 审批内容：合同细节                   │
│ ├─ 审批权限：<$20k直接批准              │
│ └─ 审批时效：2小时内                    │
├─────────────────────────────────────────┤
│ 二级审批：销售总监（条件性）             │
│ ├─ 触发条件：金额≥$20k                 │
│ ├─ 审批内容：风险评估                   │
│ └─ 审批时效：4小时内                    │
└─────────────────────────────────────────┘
```

---

### **原因2：权限分级清晰** ✅

| 角色 | 审批权限 | 金额范围 | 是否必须 |
|------|---------|---------|---------|
| 区域主管 | 一级审批 | 所有金额 | ✅ 必须 |
| 销售总监 | 二级审批 | ≥$20,000 | ⚠️ 条件性 |

**授权额度表：**
```
┌─────────────┬─────────────┬─────────────┐
│ 合同金额     │ 区域主管    │ 销售总监    │
├─────────────┼─────────────┼─────────────┤
│ $0 - $19,999│ ✅ 可直接批准│ ⏭️ 无需审批  │
├─────────────┼─────────────┼─────────────┤
│ ≥ $20,000   │ ✅ 必须审批  │ ✅ 必须审批  │
└─────────────┴─────────────┴─────────────┘
```

---

### **原因3：便于流程跟踪** ✅

**两个独立步骤的优势：**

```
审批历史记录（拆分后）：
┌────────────────────────────────────────────┐
│ ✅ 步骤12：区域主管审批                     │

│    审批结果：✅ 通过                        │
├────────────────────────────────────────────┤
│ ✅ 步骤13：销售总监审批                     │
│    审批人：王强（销售总监）                 │

vs.

审批历史记录（合并，不推荐）：

└────────────────────────────────────────────┘
```

```
│   └─ 授权额度：部门预算内               │

├────────────────────────────────────────┤
│ 层级2：销售总监（条件性）               │
│   └─ 触发条件：≥$20,000                │
└────────────────────────────────────────┘
```

## 📐 详细设计方案

### **步骤12：区域主管审批销售合同（一审）**

```yaml
步骤编号: 12
步骤标题: "区域主管审批销售合同"
执行角色: "区域业务主管"
执行人员:
  - 刘建国 (北美区域主管)
  - 陈明华 (南美区域主管)
  - 李国强 (欧非区域主管)

审批内容:
  - 合同条款是否合理
  - 价格是否符合公司策略
  - 交期是否可行
  - 付款条件是否可接受

审批权限:
  - 金额 < $20,000: 可直接批准，流程进入步骤14
  - 金额 ≥ $20,000: 需总监审批，流程进入步骤13

表单字段:
  - 合同编号: SC-2024120801
  - 合同金额: $25,000
  - 审批意见: [文本框]
  - 审批结果: [通过/拒绝]
  - 是否需要总监审批: 自动判断（金额≥$20k）

通知规则:
  - 触发人: 业务员（提交合同）
  - 被通知人: 
      - 区域主管（对应区域）
      - CC：销售总监（仅大额合同）
  - 通知内容: "业务员{name}提交了销售合同{SC号}，金额${amount}，请审批"

下一步:
  - 通过 + 金额<$20k → 步骤14（采购员拆单）
  - 通过 + 金额≥$20k → 步骤13（销售总监审批）
  - 拒绝 → 退回步骤11（业务员修改）
```

### **步骤13：销售总监审批销售合同（二审）** 🆕

```yaml
步骤编号: 13
步骤标题: "销售总监审批销售合同"
执行角色: "销售总监"
执行人员:
  - 王强 (销售总监)

触发条件:
  - 合同金额 ≥ $20,000
  - 区域主管已审批通过

审批内容:
  - 大额合同风险评估
  - 客户信用评估
  - 公司资源评估
  - 利润率评估

审批权限:
  - 最终审批权，通过后合同生效

表单字段:
  - 合同编号: SC-2024120801
  - 合同金额: $25,000
  - 区域主管意见: 刘建国已批准
  - 风险评估: [高/中/低]
  - 审批意见: [文本框]
  - 审批结果: [通过/拒绝]

通知规则:
  - 触发人: 区域主管（审批通过）
  - 被通知人: 销售总监王强
  - 通知内容: "区域主管{name}已审批销售合同{SC号}，金额${amount}≥$20,000，请二审"

下一步:
  - 通过 → 步骤14（采购员拆单）
  - 拒绝 → 退回步骤11（业务员修改）
```

### **步骤14：采购员拆单（原步骤13）**

```yaml
步骤编号: 14 (原13)
步骤标题: "采购员拆单"
执行角色: "采购员"
执行人员:
  - 刘刚 (采购员)

前置条件:
  - 销售合同已审批通过（步骤12或步骤13）

操作内容:
  - 根据SC拆分为多个采购单（PO）
  - 分配给不同供应商
  - 生成采购编号

下一步:
  - 步骤15（采购主管审批）
```
