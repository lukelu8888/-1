## 1. 目的

这份表用于把三层口径统一起来：

1. `FullProcessSandboxV5` 的完整演示流程
2. `后段蓝图 V2` 的真实业务主线
3. 当前仓库中的真实实现状态

原则：

- `V5` 用来做流程参考，不直接等同于真实 ERP
- `后段蓝图 V2` 是当前真实业务定义
- “当前实现”按三档判断：
  - `已真实可用`
  - `已显式接入（轻实现）`
  - `仍待补强`

## 5. 当前实现强项

当前真实实现已经比沙盒更贴近贸易实操的地方：

- `订舱` 与 `装柜` 已明确拆开
- `前T/T / 后T/T / L/C / D/P` 已进入控制逻辑
- `第三方仓 + 集货计划 + 待装柜池 + 分柜规划 + 单柜装载` 已成为正式模块
- `planned vs actual` 已进入装柜执行层
- `客户 / 内部 / 供应商` 三方已经开始共用同一套后段协同摘要
- 财务文件包已不再只是概念，已有真实上传、当前文件、缺失清单、最小日志
- `付款控制 / 银行交单 / 放单` 已从状态字段升级为统一规则、阻塞原因、动作时间和操作者
- 内部页已经开始形成“统一待办概览”，能同时看到履约卡点、财务卡点、归档卡点
