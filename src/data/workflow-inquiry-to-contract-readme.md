# 📋 客户询价到合同业务流程

## 流程概述

**流程名称**: 客户询价到合同流程  
**适用场景**: B2B外贸业务  
**适用区域**: 北美、南美、欧非  
**总计阶段**: 5个  
**总计步骤**: 21个  
**涉及角色**: 7个（客户、区域业务员、区域经理、销售总监、Admin财务、供应商、系统）

---

## 🎯 流程架构

### 阶段 1️⃣: 📋 询价阶段
**目标**: 客户发起询价，内部协调获取成本底价

#### 步骤流程：
1. **客户发送询价单** (Customer)
   - 填写产品需求、数量、交期、收货地址
   - 触发条件: 表单提交
   
2. **系统自动分发询价单** (System)
   - 同时通知Admin财务和区域业务员
   - 确保双方及时获知需求
   
3. **区域业务员提醒财务报价** (Regional Sales)
   - 标注紧急程度和客户背景
   - 设定期望响应时间
   
4. **财务生成工厂询价单** (Admin Finance)
   - 选择供应商列表
   - 设置响应截止日期
   
5. **工厂反馈询价单** (Supplier)
   - 提供单价、MOQ、交期、付款条件
   
6. **财务提供成本底价** (Admin Finance)
   - 汇总供应商报价
   - 计算成本底价和建议售价
   - 标注利润率

---

### 阶段 2️⃣: 💰 报价阶段
**目标**: 区域业务员创建客户报价单

#### 步骤流程：
7. **区域业务员创建报价单** (Regional Sales)
   - 基于成本底价定价
   - 必须≥成本价
   - 填写有效期、付款条件、交货条件
   
8. **金额自动路由审批** (System)
   - **< 2万美金** → 区域经理审核
   - **≥ 20万美金** → 销售总监审核
   - **2万-20万美金** → 区域经理审核

---

### 阶段 3️⃣: ✅ 审批阶段
**目标**: 多级审批确保报价合理性

#### 步骤流程：
9. **区域经理审核报价单** (Regional Manager)
   - 适用于 < 20万美金订单
   - 审核价格、利润率、付款条件
   - 决定: 通过 / 不通过
   
10. **销售总监审核报价单** (Sales Director)
    - 适用于 ≥ 20万美金订单
    - 评估风险和战略价值
    - 决定: 通过 / 不通过
    
11. **审核结果反馈** (System)
    - **通过** → 进入客户提交流程
    - **不通过** → 业务员查看原因并修改

12. **业务员查看不通过原因** (Regional Sales)
    - 查看具体拒绝理由
    - 修改报价后重新提交审批
    - 返回步骤7

---

### 阶段 4️⃣: 💬 客户反馈阶段
**目标**: 提交报价给客户并处理反馈

#### 步骤流程：
13. **区域业务员提交报价给客户** (Regional Sales)
    - 生成报价单PDF
    - 附带说明邮件
    
14. **客户收到Admin报价** (Customer)
    - 通过Customer Portal查看详情
    - 查看产品明细、价格、交期
    
15. **客户反馈报价** (Customer)
    - 三种选择:
      - ✅ **同意** → 进入合同阶段
      - 💬 **议价** → 上报区域经理
      - ❌ **拒绝** → 业务员跟进原因
    
16. **业务员处理客户反馈** (Regional Sales)
    - 根据反馈类型路由到不同流程
    
17. **区域经理处理议价** (Regional Manager)
    - 评估客户议价要求
    - 决定接受或提出折中方案
    - 返回步骤13重新提交
    
18. **业务员跟进拒绝原因** (Regional Sales)
    - 联系客户了解具体原因
    - 记录反馈用于改进
    - 评估未来机会
    - **流程结束**

---

### 阶段 5️⃣: 📄 合同阶段
**目标**: 生成销售合同并审核

#### 步骤流程：
19. **生成销售合同** (Regional Sales)
    - 基于确认的报价单生成
    - 填写合同条款、特殊条款
    - 设置付款计划、交货计划
    
20. **区域经理审核合同** (Regional Manager)
    - 审核完整性、合规性、风险点
    - 决定: 通过 / 不通过
    
21. **合同审核通过** (System)
    - 发送合同给客户签署
    - 通知财务部门
    - 通知物流部门
    - **流程成功结束**

22. **合同修订** (Regional Sales)
    - 审核不通过时触发
    - 根据反馈修改合同
    - 返回步骤20重新审核

## 🔀 关键决策点

### 金额路由决策
```
订单金额判断:
├─ < $20,000     → 区域经理审核
├─ $20,000 - $199,999 → 区域经理审核
└─ ≥ $200,000    → 销售总监审核
```

### 审批结果决策
```
审批结果:
├─ 通过   → 提交给客户
└─ 不通过 → 查看原因 → 修改 → 重新提交
```

### 客户反馈决策
```
客户反馈:
├─ 同意   → 生成合同 → 合同审核
├─ 议价   → 区域经理协商 → 重新报价
└─ 拒绝   → 跟进原因 → 记录 → 结束
```

### 合同审核决策
```
合同审核:
├─ 通过   → 发送客户 → 通知各部门 → 成功结束
└─ 不通过 → 修订合同 → 重新审核
```

## 👥 角色职责

### 1. Customer (客户)
- 提交询价单
- 查看报价
- 反馈报价（同意/议价/拒绝）
- 签署合同

### 2. Regional Sales (区域业务员)
- 提醒财务报价
- 创建客户报价单
- 提交报价给客户
- 处理客户反馈
- 生成销售合同
- 跟进拒绝原因

### 3. Regional Manager (区域经理)
- 审核报价单（<20万美金）
- 处理客户议价
- 审核销售合同

### 4. Sales Director (销售总监)
- 审核大额报价单（≥20万美金）
- 评估风险和战略价值

### 5. Admin Finance (Admin财务)
- 接收客户询价
- 生成工厂询价单
- 提供成本底价

### 6. Supplier (供应商)
- 反馈工厂询价单
- 提供报价和交期

### 7. System (系统)
- 自动分发询价单
- 金额路由
- 审核结果通知
- 合同发送

- **高级审批**: 销售总监（≥20万美金）

## 🎨 使用说明

### 如何加载此流程？

1. **方法一: 直接加载**

   - 点击"加载询价流程"按钮

   - 点击"导入JSON"按钮

3. **方法三: 手动复制**
   - 打开JSON文件
   - 复制全部内容
   - 在编辑器中粘贴

1. **修改阶段名称**: 点击阶段卡片 → 编辑名称
