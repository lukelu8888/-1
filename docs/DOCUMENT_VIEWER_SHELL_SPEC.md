# 统一查看器外壳规范

适用范围：
- 业务侧眼睛预览弹窗：`ING / QR / QT / SC`
- 采购侧眼睛预览弹窗：`QR / XJ / BJ / PR / CG`

目标：
- 所有文档预览统一使用同一套外壳，模板正文保持各自独立。
- 外壳只负责“查看”，不承载额外业务处理面板。

## 一、统一外壳

统一使用：
- `src/components/documents/StandardDocumentViewerShell.tsx`

采购侧别名：
- `src/components/admin/purchase-order/ProcurementDocumentViewerShell.tsx`

## 二、保留项

每个预览弹窗默认只保留以下结构：
- 标题
- 单号副标题
- 模板版本
- 必要徽标
- 操作按钮
- 文档正文预览区

允许保留的头部附加信息：
- 与文档本身强相关、且稳定存在的 badge
- 例如 `SC` 的付款模式、触发节点

## 三、禁止项

以下内容禁止再插入到正文上方：
- 英文说明副标题
- 付款条款说明文案
- 流程摘要条
- 当前卡点
- 时间轴
- 额外说明卡片
- 任何内部处理面板

说明：
- 这些内容如果需要展示，应放回列表、详情页或独立业务区域，不进入统一查看器正文前区。

## 四、按钮顺序

统一顺序：
1. `下载PDF`
2. `打印`
3. 其他动作
4. `关闭`

说明：
- `关闭` 由统一外壳提供。
- 模块侧不要重复放第二个关闭按钮。

## 五、副标题规则

副标题统一显示当前主单号：
- `ING` 显示 `inquiryNumber`
- `QR` 显示 `requirementNo`
- `QT` 显示 `qtNumber`
- `SC` 显示 `contractNumber`
- `XJ` 显示 `xjNo`
- `BJ` 显示 `quotationNo / bjNumber`
- `PR` 显示 `prNumber`
- `CG` 显示 `poNo`

禁止把项目编码、流程串联说明、拼接说明语放入副标题。

## 六、模板边界

统一查看器只负责模板外壳：
- 遮罩
- 弹窗容器
- 顶部工具栏
- 预览区底色与居中

模板正文内部：
- 页面结构
- 字段排布
- 分页逻辑
- 页脚页码

继续由各文档模板自行负责。

## 七、新增预览接入要求

新增眼睛预览时必须满足：
- 使用 `StandardDocumentViewerShell`
- 不使用旧的自定义 `DialogContent` 预览壳
- 不在 `children` 前后插入解释块
- 不在预览弹窗中挂业务处理组件

## 八、回归检查

每次调整预览壳后至少检查：
- `询价管理`
- `成本询报`
- `报价管理`
- `订单管理`
- `报价请求池`
- `询价管理（XJ）`
- `供应商报价（BJ）`
- `采购请求池`
- `采购订单`

检查项：
- 弹窗能打开
- 标题、副标题、模板版本正常
- 按钮顺序一致
- 正文前无额外说明块
- 模板正文正常显示
