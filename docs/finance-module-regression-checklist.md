# 财务模块回归检查清单

适用范围：
- 应收账款列表
- 收款录入中心弹窗
- 定金/余款客户上传凭证
- 财务确认到账
- 刷新后状态回显

## 1. 定金链路

前置条件：
- 合同已生成
- 客户侧可看到定金上传入口

检查步骤：
1. 客户上传定金付款凭证
2. 财务侧进入应收账款列表，确认该单进入“待财务确认定金到账”或等价状态
3. 打开详情弹窗，确认左侧“定金信息”显示客户上传凭证
4. 输入“实际到账”和“到账日期”
5. 点击“确认定金到账”
6. 关闭弹窗并刷新页面
7. 重新打开该单

预期结果：
- 列表状态变为“待客户上传余款”或后续正确阶段
- 弹窗左侧显示“定金已确认到账”
- 已确认回款金额更新
- 定金实际到账/到账日期不丢失
- 刷新后不回退为可编辑状态
- 控制台无新的 `orders` / `payments` / `accounts_receivable` 400

## 2. 余款上传链路

前置条件：
- 定金已确认到账
- 客户侧可看到余款上传入口

检查步骤：
1. 客户上传余款付款凭证
2. 财务侧刷新应收账款列表
3. 确认该单进入“待财务确认余款到账”
4. 打开详情弹窗，确认右侧“余款信息”显示客户上传凭证

预期结果：
- 列表状态显示“待财务确认余款到账”
- 弹窗右侧显示余款凭证、余款金额、汇款日期、上传人
- 刷新后不退回“待客户上传余款”

## 3. 余款确认链路

前置条件：
- 客户已上传余款付款凭证

检查步骤：
1. 财务输入“实际到账”
2. 财务输入“到账日期”
3. 点击“确认余款到账”
4. 关闭弹窗并刷新页面
5. 重新打开该单

预期结果：
- 列表显示“已结清 / 无需跟进”或符合业务规则的最终状态
- 弹窗右侧显示“余款已确认到账”或“已结清”
- 已确认回款金额等于合同总额，未回款为 0
- 刷新后仍保持已结清
- 不出现“临时成功，刷新后回退”

## 4. 手续费容差链路

前置条件：
- 客户汇款金额与财务实际到账存在小额差值
- 差值在手续费容差范围内

检查步骤：
1. 客户上传余款凭证，金额为合同余款金额
2. 财务确认到账时，输入略低于客户汇款金额的实际到账值
3. 点击“确认余款到账”
4. 刷新页面

预期结果：
- 列表显示“已结清”，而不是“部分收款”
- 弹窗顶部显示“已结清 / 无需跟进”
- 已确认回款显示合同总额
- 实际到账仍保留为真实银行入账金额

## 5. 列表与弹窗一致性

检查步骤：
1. 在列表中记录该单当前状态
2. 打开详情弹窗
3. 对比列表与弹窗的关键信息

必须一致的字段：
- 流转状态
- 已确认回款
- 未回款
- 定金阶段标签
- 余款阶段标签

预期结果：
- 不出现“列表已结清，弹窗待确认”
- 不出现“弹窗已结清，列表仍显示待处理”

## 6. 刷新一致性

检查步骤：
1. 完成一次定金或余款确认
2. 刷新页面 2 到 3 次
3. 关闭并重新打开对应弹窗

预期结果：
- 状态稳定
- 金额稳定
- 到账日期稳定
- 凭证下载入口稳定

## 7. 控制台与网络检查

检查步骤：
1. 打开浏览器控制台
2. 打开 Network 并勾选“保留日志”
3. 完成一次定金或余款确认
4. 观察是否有新的失败请求

通过标准：
- 不出现新的 `accounts_receivable` 400
- 不出现新的 `orders` 400
- 不出现新的 `payments` 400
- 不出现“写入 Supabase 失败”类 toast

备注：
- `ipwho.is` 相关日志不应再在 localhost 初始化时触发
