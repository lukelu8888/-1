# 📊 订单流程文档管理 - 紧凑表格视图（台湾大厂风格）

## 🎯 设计理念

采用**台湾制造业大厂**（台积电、鸿海、华硕等）的专业工业化设计风格：
- ✅ **高信息密度** - 在有限空间内展示最多信息
- ✅ **严谨专业** - 表格化、结构化、工程化
- ✅ **紧凑高效** - 减少视觉干扰，提升操作效率
- ✅ **层级分明** - 阶段→节点→文档三层架构
- ✅ **状态可视化** - 颜色编码、图标标识、进度条

## 🎯 核心设计特点

### **1. 阶段标题栏**
- **渐变背景** - 每个阶段不同颜色（蓝/紫/橙/绿/青/灰）
- **双层边框** - 2px粗边框 + 内边框
- **进度显示** - X/Y 已完成 + 进度条
- **折叠图标** - ▼ 展开 / ▶ 折叠
- **点击折叠** - 整个标题栏可点击

### **2. 节点表格**
| 列名 | 宽度 | 说明 |
|------|------|------|
| # | 固定 50px | 节点序号（1-16） |
| 节点名称 | 自适应 | 节点名称 + 🔗图标（如有Admin互动） |
| 状态 | 固定 100px | 状态徽章（已完成/进行中/待上传/待处理） |
| 文档数 | 固定 80px | 圆形徽章显示数量 |
| Admin互动 | 固定 200px | Admin状态（角色+人名） |
| 操作 | 固定 130px | [查看][上传] 按钮 |

### **3. 节点状态徽章**

| 状态 | 颜色 | 图标 | 背景色 | 文字色 |
|------|------|------|--------|--------|
| ✅ 已完成 | 绿色 | CheckCircle | bg-green-100 | text-green-700 |
| ⏱ 进行中 | 蓝色 | PlayCircle | bg-blue-100 | text-blue-700 |
| ⚠️ 待上传 | 橙色 | AlertCircle | bg-orange-100 | text-orange-700 |
| ⭕ 待处理 | 灰色 | PauseCircle | bg-gray-100 | text-gray-600 |

### **4. 文档数徽章**
```tsx
// 有文档：蓝色圆形徽章
<span className="w-6 h-6 bg-blue-100 text-blue-700 rounded-full">3</span>

// 无文档：灰色圆形徽章
<span className="w-6 h-6 bg-gray-100 text-gray-400 rounded-full">0</span>
```

### **5. Admin互动显示**
```tsx
// 已完成
✅ 财务 (王芳)

// 进行中
⏱ 业务员 (李明)

// 待处理
⚠️ 单证员 (张三)

// 无互动
-
```

### **6. 操作按钮**
```tsx
// 查看按钮（展开/收起文档列表）
[▼ 查看] / [▲ 收起]

// 上传按钮（蓝色强调）
[📤 上传] - bg-blue-50 hover:bg-blue-100 border-blue-300
```

## 🎨 颜色系统

### **阶段颜色**
| 阶段 | 颜色 | 背景渐变 |
|------|------|---------|
| 订单前期 | 蓝色 | from-blue-50 to-blue-100 |
| 样品确认 | 紫色 | from-purple-50 to-purple-100 |
| 生产执行 | 橙色 | from-orange-50 to-orange-100 |
| 验货包装 | 绿色 | from-green-50 to-green-100 |
| 发货收款 | 青色 | from-cyan-50 to-cyan-100 |
| 交付完成 | 灰色 | from-gray-50 to-gray-100 |

### **边框系统**
- **主边框** - `border-2 border-gray-300` (2px 灰色)
- **内边框** - `border border-gray-200` (1px 浅灰)
- **强调边框** - `border-2 border-blue-300` (2px 蓝色)

### **背景系统**
- **卡片背景** - `bg-white`
- **斑马纹** - 奇数行 `bg-white`，偶数行 `bg-gray-50`
- **悬停效果** - `hover:bg-gray-50`

## 📏 尺寸规范

### **字体大小**
- **标题** - 15px（font-bold）
- **小标题** - 13px（font-medium）
- **正文** - 12px（text-xs）
- **说明文字** - 11px（text-xs text-gray-500）

### **间距**
- **卡片外边距** - 12px (space-y-3)
- **内边距（大）** - 16px (p-4)
- **内边距（中）** - 12px (p-3)
- **内边距（小）** - 8px (p-2)
- **元素间距** - 12px (gap-3)

### **高度**
- **标题栏** - 40px (py-2)
- **表格行** - 36px (py-2)
- **输入框** - 32px (h-8)
- **按钮** - 24px (h-6)

## 🔄 交互流程

### **1. 选择订单**
```
顶部订单选择器 → 下拉选择具体订单 → 
自动加载该订单的所有节点和文档 →
显示订单金额、税务信息、进度
```

### **2. 查看节点文档**
```
点击阶段标题 → 展开/折叠该阶段的所有节点 →
点击节点"查看"按钮 → 展开该节点的文档列表 →
可预览、下载文档
```

### **3. 上传文档**
```
点击节点"上传"按钮 →
普通节点 → 通用上传对话框 →
增值税发票节点 → 专用发票上传对话框 →
选择文件 → 确认上传 → 文档添加到节点 →
节点状态自动更新（pending → in_progress）
```

### **4. 全部展开/折叠**
```
顶部"全部展开"按钮 → 展开所有6个阶段 →
顶部"全部折叠"按钮 → 折叠所有6个阶段
```

## 🚀 使用场景

### **场景1：快速浏览订单进度**
```
进入页面 → 选择订单 → 
查看顶部进度条（70%） →
扫描各阶段进度（订单前期 67%、样品确认 100%...） →
快速定位到待处理节点（发货收款 25%）
```

### **场景2：上传特定节点文档**
```
选择订单 → 展开"发货收款"阶段 →
找到"增值税发票"节点（⚠️待上传） →
点击"上传"按钮 → 填写发票信息 →
上传发票扫描件 → 提交 → 
节点状态更新为"⏱进行中"
```

### **场景3：检查Admin端处理状态**
```
选择订单 → 展开任意阶段 →
查看"Admin互动"列 →
✅ 已完成节点 - 确认无误
⏱ 进行中节点 - 等待处理
⚠️ 待处理节点 - 需要催促
```

## 🔧 技术实现

### **关键组件**
```typescript
- SupplierDocumentsWorkflowCompact.tsx (主组件)
- expandedPhases (阶段展开状态)
- expandedStages (节点展开状态)
- workflowStages (16个节点定义)
- selectedOrderId (当前选中订单)
```

### **核心函数**
```typescript
- togglePhase() - 展开/折叠阶段
- toggleStage() - 展开/折叠节点
- handleUploadToStage() - 上传文档到节点
- handleUploadVATInvoice() - 上传增值税发票
- calculateProgress() - 计算订单进度
```
